# 站点即时同步配置记录(2026-07-02)

## 做了什么

为 `.github/workflows/notify-site.yml` 配齐了缺失的仓库 Secret `SITE_DISPATCH_TOKEN`,打通「知识库 push → 展示站即时重建」链路。

- Token 类型:fine-grained PAT,仅授权 `above-the-web` 单仓库,权限仅 Contents: Read and write
- 写入方式:`gh secret set SITE_DISPATCH_TOKEN --repo Mr-Salticidae/knowledge-base`
- 生效机制:push 到 main 且改动 `0*/**` 或 `README.md` 时,向 above-the-web 发 `repository_dispatch(kb-updated)`,触发其 Build & Deploy

## 之前的状态

Secret 未配置时 workflow 优雅跳过(不报错),站点只能靠 6 小时定时兜底同步;配置后内容更新可即时上线,定时同步继续作为兜底保留。

## 维护提醒

- PAT 有效期到期后,workflow 会静默退回「优雅跳过 + 定时兜底」,站点不会坏,只是不再即时。届时重新生成 token 并再跑一次 `gh secret set` 即可。
- 本文件的提交本身即为 dispatch 链路的首次验证提交。
