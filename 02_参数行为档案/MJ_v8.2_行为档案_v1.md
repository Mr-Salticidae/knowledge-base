# MJ v8.2（--preview）行为档案 · v1

> 入档：2026-07-03
> 触发：49_凝视_写实重置 EP02 VESPER 开工，v8.2 首次实测（两批 11 张）
> 关联：[[MJ_v8.1_无oref_混合工作流]] / [[新模型试跑_硬锁清单先行_v1]] / [[角色一致性金字塔]]

## 一句话

**v8.2 走 `--preview` 参数调用（预览通道），写实人像质感较 v8.1 再上一档；异色瞳等"具体色名直给"类硬锁一句话即可稳定命中，配 sref 可跨批续接质感。**

---

## 调用方式

prompt 末尾加 `--preview` 即路由到 v8.2 预览模型（与 `--v` 写法并存期的临时通道，正式版参数待观察）。实测搭配：`--ar 9:16 --raw --hd --preview`。

---

## 实测行为（49 EP02 · 单项目样本）

### 1. 写实质感 ⚠️首次

老 prompt（半写实厚涂词系 `semi-realistic anime illustration style, painterly rendering`）直跑，出图质感即达"发丝级细节 + 真实皮肤过渡"，明显优于同 prompt 在旧版本的表现。**质感提升不挑词系**——连非写实词系都被拉高。

### 2. 异色瞳一句直给稳定命中 ⚠️首次（6/6，样本小）

```
one cold ice-blue eye and one warm rose-pink eye
```

- 写法要点：**左右 + 冷暖 + 具体色名**，不写抽象单词 heterochromia
- 单批 6/6 全中，预备的 `--no matching eye color, symmetrical eyes` 双保险未动用
- 边界：双眼均清晰可见的近景构图下验证；侧脸 / 遮挡构图未测
- 旧版本经验（异色瞳老大难、需负面词双保险 + 局部重绘兜底）在 v8.2 上**疑似过时**，待跨项目再验

### 3. sref 跨批质感续接 ⚠️首次

改 prompt 变量（本次改瞳色短语）时 sref 上一批成品图，新批与旧批**视觉无断裂**——"修 bug 不丢手感"。分工理解：sref 管质感氛围，不管脸的一致性（锁脸仍走 [[角色一致性金字塔]] 的描述锚定 / oref 路线）。

### 4. 非人像场景质感 ⚠️首次（闪电战单样本 · 2026-07-16）

「同根同源」闪电战《七步诗 · 空屋火影》：古代灶间写实场景（土墙颗粒 / 火光层次 / 蒸汽体积感 / 硬光影子）一版立住，写实底座直接撑起影子叙事的可信度。参数组合 `--ar 3:4 --preview --style raw` 可用（与本档案人像侧的 `--raw --hd` 并存，未见冲突）。影子形态指令 `shadows clearly shaped as ...` 命中但形态偏离（two little brothers → 一大一小身影），属「偏离即升级」。详见 [[2026-07-16_同根同源_七步诗空屋火影高票复盘]]。

### 5. 未测项（占坑）

- v8.2 是否支持 oref / cref（v8.1 不支持，见 [[MJ_v8.1_无oref_混合工作流]]）
- `--stylize` 在 preview 通道下的响应曲线
- 静物 / 特写题材的质感增益（场景已开测，见第 4 条）

## 换用建议

- 写实人像项目：**直接上 v8.2**，老 prompt 可直跑试水，但入库前必须过角色卡硬锁清单（踩坑见 [[新模型试跑_硬锁清单先行_v1]]）
- 需要锁脸的多镜头项目：v8.2 质感镜头 + 锁脸镜头分工的混合策略，沿用 v8.1 档案的 A/B/C 分类法
