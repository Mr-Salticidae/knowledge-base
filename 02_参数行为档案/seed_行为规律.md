# `--seed` 行为规律

> 首次实测:2026-05-01(02 教程素材生产 · 方法 B 测试)
> 入档:2026-05-02
> 主线作品来源:`02_角色一致性锚点/07_方法B_seed_prompts.md`

---

## 一句话总结

**seed 锁的不是"微调",是"同帧锁"**——94% 构图相似 / 11% 微调指令执行率。给的不是"我能改细节",是"同一帧的不同瞬间"。

---

## 如何使用

| 你的需求 | 用什么 |
|---|---|
| 我有一张完美图,想要它**多个变体** | **seed** ✅ |
| 我想要"专辑封面 / 海报"系列 | **seed**(构图统一) |
| 我的角色出现在**不同场景** | ❌ 用 oref,不是 seed |
| 我想 prompt 加披风 / 改光线 | ❌ 都不行,要重写 prompt 不依赖锁定 |

**关键**:seed 适合"完美图扩展系列变体"。不适合"角色 IP 跨场景"。

## 实测数据(4 张微调对照基准图)

| 微调指令 | 基准图相似度 | 微调指令执行率 | 实际表现 |
|---|---|---|---|
| 闭眼浅笑 | 95% | ⚠️ 10% | 还是低头忧郁 |
| 加披风 | 90% | ❌ 5% | 披风未出现 |
| 暖夕阳 | 95% | ❌ 0% | 依然冷调 |
| 低眉沉思 | 95% | ⚠️ 30% | 眼神方向略变 |
| **平均** | **94%** | **11%** | — |

## seed 锁住了什么 / 锁不住什么

**锁住的**:
- 整体构图、人物位置
- 整体调性、配色
- 雪夜氛围、汉服款式
- 镜头角度、视觉重心

**阻止的**:
- 表情翻转(忧郁→浅笑)
- 服装新增(加披风)
- 光线反转(冷雪→暖夕阳)
- 视线大幅变化(仰望→低头)

**含义**:**seed 给的是"同帧的不同瞬间"**——而"瞬间"之间能差多少,**AI 说了算,你说了不算**。

## seed 实操技巧(网页端)

### 路径 A · 从基准图抠 seed
1. 进入 Create / Archive 页面找到基准图
2. 点击图片进大图详情页
3. "..."三点菜单 → "Copy Seed"(或在 Job Info 里找 seed 字段)
4. 复制下数字

### 路径 B(更聪明)· prompt 末尾指定固定 seed
```
[完整 prompt]
--seed 88888
```
随便写一个数字(好记的如 12345 / 88888 / 1314520),MJ 用这个 seed 生成。**这条勾起的图就是你的"方法 B 基准图"**,后面跑微调时全部用同一个 seed。

> **教学价值**:这条小贴士本身就是粉丝最爱的"原来这么简单"型干货——
> *"你不用去抠 seed,prompt 末尾加 `--seed [任意 6-8 位数字]`,MJ 就会用这个 seed。"*

## seed vs oref · 决策表

| 你的需求 | 用什么 |
|---|---|
| 我的角色出现在不同**场景** | **oref** ✅ |
| 我有一张完美图,想要它**多个变体** | **seed** ✅ |
| 我想要"角色 IP 化"做长期系列 | **oref**(场景自由度高) |
| 我想要"专辑封面 / 海报"系列 | **seed**(构图统一) |
| 我想 prompt 加披风 / 改光线 | **都不行**——这种是要重写 prompt 不依赖锁定 |

## 金句段落(可直接用于内容创作)

> *"oref 是'像她'——空间任你换,人不变。*
> *seed 是'同一帧的不同时刻'——构图被锁住,只能微调表情、衣着、光线。*
> *理解这个差异,你才知道什么场景该用 oref,什么场景该用 seed。"*

> *"我以为 seed 是用来微调细节的——结果它跑出 4 张几乎一样的图。*
> *我让她改表情,她改了一点点。让她加披风,没加。让她变暖光,还是冷雪。*
> *这不是 AI 在偷懒——是 seed 在执行它真正的工作:复刻同一帧。*
> *seed 给的不是'微调',是'同帧的不同瞬间'。'瞬间'之间能差多少——AI 说了算。"*

## 升级补丁(2026-05-02 自由探索副会话二 · 胶片人像探索)

> 触发:跑"清冷东方少女回眸"时,固定 seed=42 出来只是"中等水平",解锁 seed 后**每一张都比 seed=42 那张更强**。

### 新增规律:Seed 锁定的"阶段适用性"

原档案重点讲 "seed 锁的是什么 / 锁不住什么"。这次发现另一个之前没正式讨论过的维度:**Seed 锁定不是一直该做的事 —— 只在特定阶段该做**。

| 创作阶段 | 该不该锁 seed |
|---|---|
| 测参数行为(sw / sref / 顺序) | ✅ 必锁(排除随机性) |
| 改 prompt 措辞看效果 | ✅ 必锁(排除随机性) |
| 验证某条规律 | ✅ 必锁 |
| **prompt 已定型,在找最好的产出** | ❌ **必须解锁,多 roll 几轮挑** |

### 为什么之前没意识到这条

刚结束 multi-sref 实验时记下了"做对照实验 seed 必锁"的教训(重要,正确)。但这条惯性**带到了"找最好产出"的阶段**,导致用一个固定 seed 模具压一个其实有很大表现谱的 prompt。Seed=42 不是任何 prompt 的"最佳 seed",它只是"一个 seed" —— 同一 prompt 在不同 seed 下的产出**质量谱很宽**。

### 实操心法

> **创作工作流分两阶段:**
>
> 1. **探索/验证阶段** —— 锁 seed,变其他变量,目的是排除随机性、看清因果
> 2. **打磨/挑选阶段** —— 解锁 seed,变 seed,目的是从同一 prompt 的产出谱里挑最佳
>
> **从阶段 1 切到阶段 2 的信号**:你已经不再问"这个 prompt 能不能"——你开始问"这个 prompt 能跑出多好的"。

### 一个延伸技巧

打磨阶段挑出最爱那张后,**回头读它的 seed**,然后回到阶段 1 的工作流(锁住这个 seed,微调 prompt)—— 这是从"打磨"切回"细调"的桥梁。
