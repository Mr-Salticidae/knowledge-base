# 可灵 Kling 3.0 / Omni 行为规律

> 来源:R-07 项目 2026-05-11 全片实跑(17 段视频 + 多轮迭代)

> 注意:可灵迭代很快,后续版本可能行为不同——见"如何使用"段

## Kling 3.0 vs Omni 实测差异

| 维度 | Kling 3.0 | Kling 3.0 Omni |
|---|---|---|
| 架构 | 从 2.6 演化 | 基于 O1 架构 |
| 专长 | 时序稳定性 / 多镜头控制 / 结构化场景 | reference-guided generation 更可靠 |
| 对源图一致性 | 较强 | **更强**(多元素 + R-07 视觉规范四要素 100% 锁住)|
| 适合场景 | 完全静止 / 简单微动 / 无主体或单元素 | R-07 + 蝶 + 其他活物的复杂多元素 / 长时长 |
| 默认运动倾向 | 倾向加入"自然微动" | 同样有 |
| 成本 | 较低 | 较高 |

**R-07 项目分配**:
- 3.0:S2 / S5 / S7 / S8 / S10 / S11 / S13b / S16
- Omni:S3 / S4 / S6 / S12 / S13a / S14 / S15

实测 R-07 一致性两者都能锁住。**简单镜头用 3.0 + 复杂镜头用 Omni 的混用策略可行**。

## "默认运动倾向"(关键发现)

可灵 3.0 即使 prompt 反复否定("不动 / 不扇翅 / 不转头"),仍保留某种程度的微动作为"自然感"。具体观察:

- S8 prompt:"蝶翅完全合拢且静止 15 秒,没有任何扇动" → 实测:蝶翅原地扇动(没飞起,但扇动)
- S5 prompt:"蝴蝶不落在手指上" → 实测:蝶第 2 秒落到手指上
- S2 prompt:"R-07 头部不转动" → 实测:R-07 略微转头朝向背景

→ **可灵把模型常识(蝶=必扇翅 / 头=会动 / 主体=有生气)放在 prompt 否定之上**。

## Prompt 写法的 4 条铁律

1. **One Movement Per Shot**——两个主导动作叠加 = 都做不好。一镜一个主导动作 + 微动叠加可
2. **慢动作 >> 快动作**——`slow / very slow / gentle` 修饰下显著优于 `fast / rapid`
3. **Positive 描述为主**——告诉模型"什么动 + 怎么动",不堆砌"不能 X / 不能 Y"
4. **接受默认微动**——小幅蝶翅扇 / R-07 微转头 / 镜头微漂移都可接受,不为这些重跑

## 核心红线(必须 100% 守住)

可灵默认微动可接受,但**这 3 条不可妥协**:
1. **角色一致性 100%**(R-07 视觉规范 / 主体形态 / 配色比例)
2. **镜头方向不能反**(写"向左"别跑成向右)
3. **主体不能跑出画面外**(蝶不能飞走 / R-07 不能瞬移)

## 推进镜头的"背景幻觉"风险

镜头推进(zoom-in)会让画面中央元素被"清晰化"。如果源图背景有**模糊轮廓 / 阴影 / 残留形状**,可灵会倾向把它"补完"为人形 / 角色。

R-07 案例 S13b v1:源图中央背景有一个阴影,镜头推进时被渲染成了"角色"。

**应对**:
- 推进镜头前,先检查源图中央是否有可疑阴影 / 模糊轮廓
- 如有,改用"横向平移"代替推进(横向不放大中央 = 阴影不被清晰化)
- 或在 prompt 里显式约束"背景的远处元素和模糊阴影保持模糊,不被识别为人形"

## 长镜拼接策略(单段 15s 上限)

- ≤ 15s 镜头:一镜到底
- > 15s 镜头:两种方案
  - **方案 A**:两段独立跑 + 剪辑硬接(保留两段精调图细节)
  - **方案 B**:第一段跑完导出末帧 → 作第二段首帧(完全无缝衔接,但末帧需要可灵生成的画面而非 MJ 精调图)
- **首尾帧上传**:可灵 UI 支持同时上传起始帧 + 结束帧,Kling 自动插值。**风险**:中间过程由可灵生成,主体"从有到无"的过渡容易翻车——不推荐用于"R-07 走出画面"这种位置变化场景

## 如何使用

- 写可灵 prompt 前打开本文件,过一遍"4 条铁律 + 3 条红线"
- 接手新 R-07 类项目时,先按"我的预设"保存 5 个常用组合(完全静止 / 微动静立 / 行走 / 拉远 / 飞行)→ 节省每次手动选词时间
- 可灵 UI 字段在新版本可能变动——如果跑出来跟本文件描述不符,**第一反应是 UI 更新了,不是模型变了**
