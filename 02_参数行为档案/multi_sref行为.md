# `multi-sref` 行为规律

> 首次实测:2026-05-02(休息日自由探索副会话)
> 入档:2026-05-02
> 触发:MJ 官网某图作者用了双 sref,惊艳出图,引发探索
> 配套素材:`{AIGC工作站}/临时创作\2026-05-03_multi_sref探索\素材图\`

---

## 一句话总结

**Multi-sref 不是"两个风格平均化",是"分层叠加"** —— 抽象风格层在低 sw 已渗透,具象元素(发饰/UI/装饰)在高 sw 才硬复刻。冲突时主体层"和 prompt 同族者赢",但败者的"伴生元素"仍会渗透。

---

## 如何使用

| 你的需求 | 怎么配 multi-sref |
|---|---|
| 想用一张 sref 但氛围更浓 | 单 sref + sw=100 默认就够 |
| 想稳定的"配方化"古风/写实人像 | 双同族 sref + sw=800 + seed 锁定 |
| 想让某个具体配饰复刻进来(发饰/裙边纹路) | 用一个含该配饰的 sref + sw=800(具象层在 sw 高时才出来) |
| 想糅合两个不同风格 | 慎用,容易出"分层割裂"图(不漂亮但有意思) |
| 想测 sref 真实风格 | **必须关掉 moodboard**,否则它会污染数据(湿发其实是它贡献的) |

**通用法则**:**双 sref 比单 sref"信号更强"** —— 估算上,同 sw 下叠加效果约等于单 sref 的 1.5-2 倍力度。

## 已验证的 6 条规律

### 规律 1 · 顺序无关(高把握)

**实验**:同 prompt + 同 seed=42 + 同 sw=800,基准图1+基准图2 vs 基准图2+基准图1

**结果**:几乎一模一样,差异仅在像素噪声级别(纱质褶皱微差)。

**含义**:MJ 处理 multi-sref 是 order-invariant 算法 —— 独立编码每张 sref 后再合并,不存在"前面的 sref 占主导"。

**实操**:写 prompt 时不用纠结 sref 先后。

---

### 规律 2 · sw 升高 → 具象元素硬复刻(高把握)

**实验**:同 prompt + 同 seed=42 + 双 sref(基准图1+基准图2),sw=400 vs sw=800

**结果**:
- sw=400:头顶金色发饰几乎不可见,只有几缕细金线
- sw=800:**头顶金色发夹清晰可辨**,基准图1 那个金色发饰风格被复刻

**含义**:Sref 信号分两层 ——
- **抽象风格层**(肤质/色调/氛围)在低 sw(默认 100)就渗透了
- **具象层**(发饰/装饰/UI 元素)在高 sw(800+)才硬复刻

**实操**:想要"sref 的具体配饰跟过来" → 拉到 sw=800;只想要"sref 的色调氛围" → sw=100 默认就够。

---

### 规律 3 · sw 还会改变 pose 选择(中等把握,需更多种子验证)

**实验**:同 sref + 同 seed=42,sw=400 vs sw=800

**结果**:同 seed 下,**两张产出明显不同的 pose**。sw=400 是侧身扭头高傲款,sw=800 是更正面柔和款。

**含义**:sw 不只是控制风格强度,也会推动 MJ 选择不同构图骨架。

**实操**:用 sw 做"风格阶梯实验"时,别期待"只有质感变化" —— 构图也会跟着动。

---

### 规律 4 · prompt 硬约束的"守不守得住"取决于风格族距离(高把握 + 反直觉)

之前一度以为"prompt 硬约束在 sw=800 内必然守得住" —— **这条结论错了**。

**实验**:基准图1 + 5692463053(navy 现代极简,风格族严重背离 prompt 的"古风金纹"指令)

**结果**:
- 单 navy 极简 sref + sw=800:服装色被吃成 navy 蓝,prompt 的 "dark gray with gold patterns" **完全失守**
- 双 sref(基准图1 + navy)+ sw=800:服装色守回深色金纹,prompt 守住了

**含义**:**Prompt 硬约束只在 sref 风格族和 prompt 兼容时守得住;风格族强烈冲突时 sref 可反吃 prompt 的硬要素。**

**实操**:
- 同族 sref 配对(都是古风、都是写实)→ prompt 安全,放心拉 sw
- 跨族 sref 配对 → prompt 的色彩/服装可能被 sref 反吃,得做好心理准备

---

### 规律 5 · sref 编码的不只是"风格" —— 还有"伴生元素"和"主体原型倾向"(高把握 + 反直觉)

这是这次最反直觉的发现。

**实验**:单 sref 5692463053(navy 现代极简) + sw=800

**结果**:
- 输出图里出现了**对话框气泡**(prompt 完全没要求)
- 人物变成了**欧裔脸**(prompt 完全没指定人种)

**含义**:**Sref 是"风格 + 伴生 UI/装饰元素 + 主体原型倾向"的复合容器,不是纯抽象风格。**

具体来说:
- 5692463053 的训练池里可能有一批"现代极简平面设计"图,带对话框/字幕条等 UI 元素 → 这些元素会"漏"到产出
- 它的训练池里的人物倾向是欧裔现代女性 → 这个偏向也会漏出

**实操**:
- 选 sref 时,**不要只看色调和氛围 —— 还要看它"会不会偷偷带主体偏向和装饰元素过来"**
- 如果 prompt 主体没强力指定(比如没明确写"亚洲女性"),sref 的主体偏向就会主导
- 高 sw 时,sref 的"伴生 UI/装饰元素"会浮现出来,这是污染源

---

### 规律 6 · 冲突 sref 的"分层仲裁"(高把握)

**实验**:基准图1(雪景古风) + 5692463053(navy 现代极简) + sw=800

**结果**:
- **主体层**(人种、服装色):由更接近 prompt 风格族的**基准图1 主导**(亚洲古风脸 + 深色金纹礼服)
- **伴生层**:5692463053 的**底部文字条仍然漏出**(没被完全压制)

**含义**:**冲突态不是赢家通吃,是分层裁决** ——
- 主体层"和 prompt 同族者赢"
- 但"败者的伴生元素仍渗透"

**实操**:**避免"风格族严重冲突的 sref 配对"**,除非你想要这种"分层割裂"的实验性效果。

## 方法论收获

### 1 · Moodboard 会污染 sref 实验

之前的双 sref 实验(带 moodboard)里,我以为基准图2 的"湿发"特征被传过来了。
关掉 moodboard 重跑后,湿发**消失了** —— 它其实是 moodboard "清冷古风" 在贡献,不是 sref。

**含义**:做 sref 实验时**必须关掉 moodboard**,否则会把它的功劳算到 sref 头上。

(这条是 [[personalize与moodboard分工]] 的具体案例验证 —— moodboard 作为"题材绑定的肌理参考池"贡献质感细节,这次"湿发"就是它的功劳。)

### 2 · Web UI 会静默吃掉 `--sw` 参数

我用的 web UI 在带 moodboard 时**静默吃掉了 `--sw` 参数**,产出的图根本没用 sw=400/800 跑。
关掉 moodboard 后 sw 才生效。

**含义**:每次写完 prompt,**要在结果参数 tag 里检查所有参数是否被接住** —— 不要假设 prompt 文本里写了就一定生效。

### 3 · 没固定 seed 跑对照实验等于送命

前两轮实验没锁 seed,种子噪声把真信号淹了 —— 一度做出错误结论("湿发 ∪ 形非单调 sw 响应"),后来发现是种子方差。

**含义**:**做对照实验,seed 必锁**(任意数字都行,固定就好)。

## 待验证的悬念(下次实验候选)

- sw 中段(300-500)是否真存在"两边都不主导"的不稳区?(妆容/姿态方差比 sw=800 更大?)
- 同向 sref 重复(`--sref A A` vs `--sref A`)是否会"双倍加深"?能验证"独立编码 + 合并"假设的强度
- "温和冲突"(同族不同风格的两个 sref,如基准图1 + 5840224573 少女水墨摄影)的调和方式
- Sref 的"主体原型倾向"是不是普遍现象?需要测多个 sref 看哪些有强主体偏向、哪些是纯风格
- 三个 sref 同时叠加是否依然 order-invariant?

## 顶层金句段(可回收)

> *"Sref 不是'纯风格容器' —— 它会把伴生 UI 元素、主体原型倾向也一起带过来。*
> *所以选 sref 时,不要只看色调和氛围 —— 还要看它'会不会偷偷带主体偏向和装饰元素过来'。"*

> *"双 sref 信号叠加,不是平均化,是分层叠加 —— 抽象风格层在低 sw 已经渗透,具象层(发饰、装饰、UI 元素)在 sw=800 才硬复刻进来。*
> *这意味着 sw 不只是'强度旋钮',它是'信号层级旋钮'。"*

> *"做 sref 实验,seed 必锁 —— 不然你以为发现的是规律,其实是种子噪声在表演。我今天就刚踩过一次。"*
