# 平台工程索引

> 用途:搭建/维护知识库展示站与各类内容平台过程中,沉淀的工程方法、架构模式与踩坑。
> 与 AIGC 创作线并列——这里记的是「怎么把东西搭起来、部署上线、可持续维护」,不是「怎么创作」。

## 架构与模式

- [[内容子项目构建时同步_独立仓库镜像进public_v1]] —— 独立频繁更新的内容仓库:浅克隆→镜像进 public(排.git)→新内容自动上线、主仓不胖
- [[CI从Release抓二进制托管自有服务器_桌面应用国内直连下载_v1]] —— CI 从 GitHub Release 抓 exe 进 dist→rsync 到香港服务器,给桌面应用做国内直连、零维护下载;二进制不进 git/Pages,固定文件名自动同步最新版
- ⚠️ [[每日定时任务自动更新内容板块_首次自动化生产落地_v1]] —— 首次自动化内容生产:定时任务只被允许「新增一个数据文件」,人先手动跑通一遍再交给定时器,并留「宁缺毋滥」不发闸门;后续已迁移云端并首次跑通,见连环坑复盘续篇
- [[站内新版块五件套模式_三次复用与起刊两坑_v1]] —— 新增内容版块的固定套路(数据源/内容集合/列表详情页/首页速览/全站导航),第三次复用(任务书版块)收敛;附 YAML 裸日期解析成 Date、机器人仓库 push 竞争两坑;07-24 补「状态机演进」:enum 扩容时取反式过滤(`!== 'open'`)会静默吞掉新状态,分桶一律正向枚举 + 新态穿线所有渲染点

## 踩坑与验证陷阱

- ⚠️ [[本地部署FastAPI静态站到3000端口_复盘_v1]] —— 本地把 FastAPI + 静态站部署到 3000 端口的复盘:先看启动脚本和依赖，再把端口改成环境变量驱动，最后用 HTTP 200 验证访问链路
- ⚠️ [[云端定时内容生产连环坑复盘_全绿不等于已发_v1]] —— GitHub Actions 定时内容生产 6 连坑(schedule 不触发/OIDC/GitHub App/工具白名单/脏工作区/中转余额)与首次真实跑通;沉淀「全绿≠已发」「手动跑通会掩盖自动路径」「黑盒 action 先接可观测性」「换便宜模型≠更省」四律
- ⚠️ [[双部署目标的base路径陷阱_根路径拼出双斜杠_v1]] —— `${BASE}/${slug}` 在根路径拼出 `//slug` 协议相对 URL 致全站内链失效;只在一个镜像现形
- ⚠️ [[导出运行时无系统字体回退律_CJK豆腐块_v1]] —— 桌面靠 OS 字体隐式补缺字形,Web/嵌入式导出无系统字体可借,拉丁字体画的中文全变豆腐块(桌面自测永远测不出);修法=给主力字体显式挂 `fallbacks` 把兜底固化进产物,零体积代价;附 headless 报错无细节须回编辑器 GUI 定位 + ETC2/ASTC 两坑(LastStand→Toy 验证导出实录)
- ⚠️ [[Claude预览环境不派发滚动事件_滚动类功能无法行为验证_v1]] —— 预览 iframe 不派发 scroll/IO 事件 + 冻结动画时钟,滚动/动效类无法行为验证;退到验证公式/逐帧采样+真机抽测;07-16 补:交互面板 rAF=0(Konva 动画/hit 图/合成双击连环失灵)→ puppeteer 无头 Chrome 三层退级已产线化
- ⚠️ [[B站Toy同步事故复盘_版本指纹与外部cron兜底_v1]] —— B站 Toy「无法同步更新」三环断裂(校验器误报/无 update 链路缺 id/线上停 v1)诊断实录;沉淀「版本指纹先定性」「校验器 ERROR 先复现解析逻辑」「沙箱 iframe 外部协议静默失效」「多镜像取最新」四条 + 「schedule 不可靠」升 ⭐⭐ 二次验证(外部 cron 兜底首跑即补救实录)
- ⚠️ [[二维码载荷硬预算与最长真实载荷验证_v1]] —— 扫码 URL 每字符都占码物理密度(中文 deflate 压缩率低,叙事全文是底盘);可选字段按硬预算逐级降配+码框位按长度自适应兜底;验证必须用最长真实载荷,调试捷径的短数据会系统性掩盖超限 bug(镜像自我扫码翻车实录)
- ⚠️ [[OpenAI兼容止于对话端点_多提供商视频接口分流与真key首测_v1]] —— 「OpenAI 兼容」只兼容到 `/chat/completions`,视频端点三家三套(硅基 `/video/submit`、方舟 `/contents/generations/tasks`、OpenAI `/videos`)需按 Base URL 分流;模型名不许猜要拉列表,且 `GET /models` 默认列表常带分类过滤(视频模型要 `?type=video`);附 flex 列压缩子项、可滚动面板内浮层被裁、错误文案截断掩盖诊断三坑(LibreCanvas 真 key 首测实录)
- ⚠️ [[OpenAI区域封锁与Worker就近执行陷阱_北美DO跳板_v1]] —— OpenAI 区域封锁跟调用方 IP 走,CF Worker 就近执行等于没挂、Smart Placement 不救,钉区域唯一免费原语是 locationHint 的 Durable Object;含公开站零密钥同域中转的分层安全闸清单与「双端对照 10 分钟定性」迁移判据(镜像自我线上真 AI 收口实录)

## 网页视觉与动效

- 🚧 [[网页主图logo流动动效迭代_液态变形与碎裂问题_v1]] —— logo「整体液态流动」正解=feTurbulence+feDisplacementMap;当前 baseFrequency 偏高致「画面太碎」,待降频减octave(周一续修)。含抠图保色/contain自适应/SPA静态资源/reduced-motion豁免等可复用要点
- ⚠️ [[reduced-motion本机陷阱_动效降级纯淡入而非跳过_v1]] —— Windows 关「动画效果」→全浏览器上报 reduce→尊重规范的动效本机全跳过,作者自测"毫无变化"(二次踩坑升律);正解=降级纯淡入(opacity 无位移安全)而非禁用;附滚动渐现完整模式(rv-on 渐进增强/animationend摘类还hover/IO错峰)+ headless 验证三坑(`--blink-settings=prefersReducedMotion=false`/虚拟时间口径/CSS被抽外部文件)
- [[文档密集页两栏排版与对外截图脱敏_v1]] —— 图多说明页塞窄栏竖堆致「又长又窄」,正解=加宽容器+图文左右两栏(`minmax(0,…)`+`align-items:start`)+窄屏回落单栏;附三纪律:对外截图「裁窗口外围>逐处打码」(authCode/组织名/内部wiki三类泄露点)、空状态给成型卡片、无头 Chrome 截图做静态视觉自检
