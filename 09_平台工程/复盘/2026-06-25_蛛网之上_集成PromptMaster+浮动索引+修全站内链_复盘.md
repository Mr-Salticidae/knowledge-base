# 2026-06-25 蛛网之上 · 集成 Prompt 大师 + 左侧浮动索引 + 修全站内链 复盘

> 入档:2026-06-25
> 站点:tiaozhuxiansheng.com(蛛网之上,Astro 静态站,Pages + 香港服务器双部署)
> 本次三件事全部上线验证通过

## 做了什么(事实)

1. **拉取更新**:本地 above-the-web 落后 6 提交,`git pull --ff-only` 快进(《反诈柜台》托管 /fraud-desk/、首页 Plays 版块、昼夜主题等)。
2. **修全站内链 bug**:用户反馈自定义域名站「关联文档」链接点击跳错。定位为根路径部署时 `${BASE}/${slug}` 拼出 `//slug` 协议相对 URL。归一化 base 前缀修复,上线后线上双斜杠内链 0 个。→ [[双部署目标的base路径陷阱_根路径拼出双斜杠_v1]]
3. **集成「成为 Prompt 大师」为独立版块**:独立仓库 becoming-a-prompt-master(自包含画廊,45MB,频繁加期)→ 构建时浅克隆镜像进 `public/prompt-master/`,首页加独立版块展示最新 4 期封面 + 头部导航。→ [[内容子项目构建时同步_独立仓库镜像进public_v1]]
4. **首页左侧浮动版块索引**:圆点导航条,hover 展开标签、滚动高亮当前版块、点击纯导航(保留全部可见,仅目标被筛选隐藏时才重置)。≤1080px 隐藏。用户反馈「体感非常好」。→ [[Claude预览环境不派发滚动事件_滚动类功能无法行为验证_v1]]

## 卡点与判断(高价值)

- **#1 不是站点没更新,是底层没更新**:用户先报「最新 skill 调整没上线」。逐层查证(远端 knowledge-base / 本机两个副本 / 全盘 grep)→ 那版「表现力/不固定构图」改动**根本不在任何仓库**,是在另一台机器没 push。**先确认内容在不在源头,再怀疑同步链路**,省掉瞎查站点的时间。
- **链接 bug 只在一个镜像现形**:Pages 子路径镜像正常,根路径镜像全废。差点因「Pages 上没问题」误判。教训:多 base 部署必须两个镜像都测。
- **浮动索引滚动高亮在预览里测不了**:Claude Preview 不派发任何 scroll/IO 事件。退到「验证判定公式 + 信任标准 API + 真机抽测」,没有因「预览无反应」把好代码改坏。

## 验证方式(本次有效的)

- 线上链接:`curl + grep 'href="//'` 计数双斜杠内链。
- 集成产物:`ls dist/prompt-master`、`.git` 未泄漏、首页 series-card 计数、封面 `naturalWidth>0`。
- 浮动索引:DOM 验布局不重叠、`compute()` 在各滚动位置返回正确版块、点击两种情况(有/无筛选)行为正确。
- 截图对 canvas rAF 页超时 → 全程用 DOM/eval 验证替代。
