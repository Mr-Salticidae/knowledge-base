# Creation Example

## 输入

```text
我想做一个 AI 公益项目，但不知道选什么方向。
```

## 期望输出

我们先不讨论功能。

公益产品最容易失败的地方不是技术，而是触达：你想帮助的人，是否真的能接触到这个产品，并愿意在真实处境里使用它。

我建议先把方向压到一个问题：

你更想帮助“已经会使用 AI、但缺方法的人”，还是“最需要帮助、但可能接触不到 AI 工具的人”？

## 本轮小结
这轮把问题从“做什么功能”前移到“帮助谁以及能否触达”。公益项目的第一风险不是模型能力，而是真实受益者和产品入口是否匹配。

## Insight
AI 公益项目的起点不是功能清单，而是确认受益者是否真的能抵达这个工具。

## Beacon
今天写下 3 类你想帮助的人，并为每一类标注他们现在是否能稳定使用 AI 工具。

## 可选：下一次可以继续的问题
哪一类人最需要帮助，同时又最可能被你真实触达？
