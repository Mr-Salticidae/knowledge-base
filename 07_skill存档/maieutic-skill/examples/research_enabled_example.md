# Research-Assisted Example

## 输入

```text
帮我找几个适合做 Claude Skill 的学习类案例。
```

## 期望输出结构

```markdown
## 信息收集结果

### 关键发现
- 学习类 Skill 适合聚焦一个可重复场景，而不是泛化成完整课程平台。
- 好案例通常有明确触发词、具体工作流、少量示例和可验证输出。
- Skill 应把长期背景放在引用材料里，入口文件只保留执行规则。

### 可能有用的资料
- 官方 Skill 创建指南
- 已有 prompt 优化类 Skill
- 已有工作流类 Skill

### 与你当前问题的关系
- Maieutic 可以参考“模式分类 + 输出模板 + 案例测试”的结构，而不是做成百科式学习助手。

## Maieutic 视角
这里真正要验证的不是“学习案例够不够多”，而是 Maieutic 能不能把一个模糊学习愿望转成下一步可行动的问题。

## Insight
适合 Maieutic 的学习类 Skill 不是课程生成器，而是一个帮助使用者定位学习入口的对话框架。

## Beacon
今天选 1 个学习场景，写出它的触发句、Maieutic 应进入的模式、以及一个 24 小时内可完成的 Beacon。
```

## 注意

实际执行时必须根据当时可用工具收集真实资料；不要把这个示例当作已完成的信息来源。
