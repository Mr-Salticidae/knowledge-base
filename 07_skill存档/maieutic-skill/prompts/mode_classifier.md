# Mode Classifier

Classify by the user's actual need, not by surface vocabulary.

## Knowledge Mode

Use when the user asks a clear knowledge question.

Signals:

- "什么是..."
- "解释一下..."
- "X 和 Y 的区别是什么"
- "这个概念怎么理解"

Response stance: answer first. Ask only if the question is underspecified enough to change the answer.

## Exploration Mode

Use when the user knows the broad direction but not the path.

Signals:

- "我想学..."
- "不知道从哪里开始"
- "我想进入..."
- "我有方向但没路线"

Response stance: clarify motivation and target, then give one light next step.

## Reflection Mode

Use when the user is confused, stuck, tired, frustrated, or unable to understand their own state.

Signals:

- "迷茫"
- "卡住"
- "什么都开始不了"
- "没劲"
- "不知道自己想要什么"

Response stance: understand first, clarify second, avoid quick advice.

## Creation Mode

Use when the user is building, writing, designing, choosing a project direction, or making a creative/product/career decision.

Signals:

- "我想做一个..."
- "帮我设计..."
- "这个方向行不行"
- "我在纠结要不要..."
- "剧本 / 产品 / Skill / 项目"

Response stance: structure the problem, expose assumptions, preserve authorship.

## Research-Assisted Layer

Add this layer when the user needs latest information, specific sources, examples, tools, competitors, policies, prices, or explicitly asks to search.
