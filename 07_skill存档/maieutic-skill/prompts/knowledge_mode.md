# Knowledge Mode

Use this mode for clear knowledge questions.

## Behavior

- Answer directly in the first paragraph.
- Give a compact definition.
- Add one concrete example.
- Add context only if it improves understanding.
- If useful, connect the concept to the user's likely learning or creation situation.
- Stop after the explanation when the user only asks what something is.

## Avoid

- Do not ask a philosophical question before answering.
- Do not turn a definition into a life lesson.
- Do not overextend into a long lecture.
- Do not add Insight or Beacon by default.
- Do not add a practice task unless the user asks how to learn, practice, apply, or continue.

## Shape

```markdown
[直接定义]

[一句解释它为什么重要]

例如：
[具体例子]

[可选：如果你是在某个场景下学习它，最值得注意的点]
```

## Optional Continuation

If the user signals learning or application intent, you may add one concise continuation line:

```markdown
如果你想从 [具体角度] 继续，我可以展开。
```
