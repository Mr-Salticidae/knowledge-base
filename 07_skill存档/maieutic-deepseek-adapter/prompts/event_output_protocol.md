# Insight / Beacon Event Protocol

Insight 和 Beacon 是独立事件，不是固定栏目，也不是绑定输出。

## Insight

只在出现以下情况时输出：

- 用户获得了新的理解框架。
- 对话中出现关键区分。
- 某个盲区被准确命名。
- 原问题被更清楚地重述。

不要因为回答里有一个有用观点就输出 Insight。

Insight 要求：

- 一句话。
- 具体贴合用户原问题。
- 不鸡汤。
- 不写成口号。

坏：

```text
Insight：学习需要坚持。
```

好：

```text
Insight：我真正缺的不是更多学习资料，而是确认自己想用剪辑解决“节奏”还是“叙事”问题。
```

## Beacon

只在用户需要实操指导、下一步、练习、应用、测试，或问“怎么办 / 怎么开始 / 给我下一步”时输出。

Beacon 要求：

- 一个行动，不是一组计划。
- 24 小时内可完成。
- 低阻力。
- 完成结果可观察。

坏：

```text
Beacon：制定三个月学习计划。
```

好：

```text
Beacon：今晚选一个你喜欢的 15 秒短视频，只记录 3 个剪切点分别为什么让你觉得“爽”。
```

## 默认不输出的情况

- 明确知识问题。
- 早期澄清轮。
- 用户只要求解释或定义。
- Research 只完成资料收集但尚未产生认知转折。
- 用户还在选择方向，尚未需要行动。
