# AIGC Poster Layout

## 适用场景

当用户要把一张已经满意的 AIGC 作品做成宣传海报、视频封面、分享图、发布预告图时使用本 skill。重点是保护原图的视觉资产，而不是重新生成一张“更像设计模板”的图。

典型触发语：

- “把这张图做成 B 站封面 / 宣传海报”
- “做一张给粉丝看的分享图”
- “不要改人，只做排版”
- “这张是核心锚点，帮我做发布图”
- “不要 PPT 感，不要流程图”

## 核心原则

1. **先保人，再做设计**：角色 IP 图里，“还是不是她/他”高于海报感。
2. **原图比例优先**：不要默认改成 16:9。先判断原图的比例是否就是气质来源。
3. **少字即传播**：粉丝向宣传图只保留角色名、情绪钩子、极小系列信息。
4. **利用原图暗部和留白**：文字应该长在图里，不要做硬切信息栏。
5. **手工排版优先**：核心锚点图不轻易交给图像模型重绘主体。

## 工作流程

### 1. 先做用途判断

明确这张图属于哪一种：

- 粉丝宣传图：追求第一眼情绪和角色吸引力。
- 平台封面图：追求缩略图可读性。
- 项目说明图：允许流程和信息密度。
- 知识复盘图：允许结构化文字，但不适合作为粉丝封面。

如果用户说“给粉丝看”“发 B 站”“视频宣传”，默认按粉丝宣传图处理。

### 2. 评估原图优势

观察原图优势来自哪里：

- 近景脸部：优先保留原比例，不要先横裁。
- 姿态和身体线条：保留全身或半身空间。
- 场景叙事：可以考虑横版扩展。
- 留白天然存在：文字放入留白。
- 没有留白但有暗部：用轻渐变压出文字空间。

不要机械套用平台比例。平台适配版可以作为第二输出，不应覆盖主视觉版。

### 3. 定文字层级

粉丝宣传图**最多两层**（默认更少）：

```text
第一层：角色名 / 作品名
第二层：一句情绪钩子（≤2 行；磁场感题材宁可只留作品名）
```

**封面不挂运营信息**——以下一律不上封面，只写进 README / 简介：

- 作者 ID / 昵称（如「跳蛛先生」）
- 系列名 / 栏目名（如「即兴抽卡」）
- 期号 / 编号 / 日期 / 署名

> 用户明确反馈（2026-06-23）：封面上不要出现作者名、栏目名这类信息；文字越少越好，编辑/时尚向尤其要克制（栖白案例：报头 + 两行封面语 + 作品名即够）。

避免这些词：制作中、工作进度、流程、已完成、归档、版本、prompt、moodboard、face locked。这些词可以写进复盘，不要写在粉丝封面上。

### 4. 排版规则

- 文字避开眼睛、鼻梁、嘴唇、脸部高光。
- 只在自然暗部、头发暗部、背景虚化、前景花影里放字。
- 不做硬分屏，不做独立黑色信息栏。
- 不使用图标、箭头、流程节点、贴纸、UI 卡片。
- 标题可以竖排，但要贴合画面气质；不要像菜单或公文标题。
- **封标设计语言要随题材切换，别一套模板套到底**：
  - 古风 / 国风：宋体（或思源宋体）竖排 + 楷体诗句 + 暖金细线；色走米白 / 暖金 / 灰绿。
  - 暗调戏剧 / 哥特：宋体竖排 + **朱红**细线 / 朱砂印章 + 角落柔暗护主体；弃暖金，用血红 + 大留白造张力。
  - 时尚 / 编辑 / 都市：仿杂志报头——**Playfair/Didot 系高反差衬线报头置顶**（可半隐于发丝），衬线报头 + 无衬线封面语（Didone 小字会断笔）。
  - 二次元 / 仙侠 cosplay / 高调薄雾：冷调空灵——冷墨蓝宋体竖排 + **冰蓝**细线（呼应蓝瞳/刃光），落进薄雾留白。
  - 梦核 / 西幻精灵 / 空灵romantic（浅暖底）：**编辑式珍珠留白**——**珍珠白仿宋**竖排主标（细劲、大字距/大行距）+ **香槟玫瑰金**1px 细线 + **Playfair 小号拉丁落款**；浅字靠左上一块**暖 aubergine 柔暗角**托出（不破高调）。蝶隐案例（2026-06-24）。
  - 脚手架（cover-fit / 柔投影 / 2x 超采样）可复用，只换文本层。
- **高饱和实心彩色衬线主标 = 廉价信号，空灵/浅梦核题材尤其忌**：用户反馈（2026-06-24）初版「葡萄紫(74,52,78)宋体实心填充」发闷发脏、字形方正死板，压在空灵图上像「AI 套色」。修正方向有二，可叠加：① **降饱和、走 tone-on-tone**——高饱和紫/蓝 → **珍珠白 + 香槟金**低饱和奢色，呼应原图肤光而非硬撞色；② **细劲字形替方正填充**——`simfang`（仿宋，细劲优雅，适合梦核/留白）或 `msyhl`（雅黑 Light，现代极简）胜过 `simsun`（宋体）的方块实心；再加大字距/大行距，留白本身即高级感。判据：缩略图像「杂志内页」而非「套了滤镜的截图」。
- **高调/近纯白底要反过来用深字**：浅底上浅字/暖金会糊，改用冷墨蓝/深字，并在字下垫一层**白色柔光晕**（由文字 alpha 染白 → 高斯模糊 → 垫底）+ 极淡冷投影做分离；别盲目套深底封面的浅色标题。
- 字体落坑：辅助黑体优先 `msyh.ttc`（微软雅黑）；**`HYZhongHeiTi-197` 缺大量中文字形会出豆腐块**。Windows 无 Didot，Playfair Display 是 OFL 可 `curl` 直拉，随项目存 `_fonts/`。空灵/留白主标用 `simfang.ttf`（仿宋·细劲），极简现代用 `msyhl.ttc`（雅黑 Light）；二者均比 `simsun` 方块填充更耐看。
- 颜色：默认米白；按题材加单一强调色（暖金 / 朱红 / 暖陶土），避免亮黄、霓虹、纯白大面积刺眼。

### 5. 输出自检

交付前必须看图自检：

- 缩小后还能第一眼看到脸吗？
- 文字有没有遮住最美的五官和光影？
- 是否像海报，而不是 PPT 或汇报页？
- 是否仍然是同一个角色？
- 字数是否还能再少？
- 成品文件名是否明确标注 FINAL 或用途？

## 工具选择

优先级（按”是否锁人脸 + 是否要批量”分流）：

1. **选帧 + GPT Image 2 叠排（图生图，**不重绘主体**）**：**单组精修封面的首选。** 上传已定稿底图，严令”人物面部/姿态/关键道具全部不变、不重绘、不美颜，只叠加排版”，让 GPT 只做标题字 + 细线 + 印章 + 柔暗角。实测**字与画面的融合、留白、质感优于代码 PIL 贴字**——代码法的字像”盖上去的”，GPT 法的字像”长在画里”。交付物是**可粘贴的 prompt**（见下节），由人在 ChatGPT 出图、成图回存。
2. 手工排版（PS/Figma 手动）：适合已定稿核心图、角色锚点图里、对单像素位置有强要求的版。
3. 代码排版（PIL/脚本贴字）：适合**批量封面、重复模板、确定尺寸**（一次出 3:4 / 4:3 / 16:9 多比例），或无法接图像模型时的稳定兜底；单张精修质感不及第 1 项。
4. Midjourney：适合重新出主题图，不适合直接给定稿图加字。

**关键区分**：被禁止的是”图像模型**重绘主体**导致人脸漂移”，**不是**”图像模型叠排版”。当用户要求”不要改人”时——① 不要让 MJ / 文生图重新生成人物；② 但**可以**用 GPT Image 2 的图生图模式给定稿图叠字，只要 prompt 里把”不重绘人物/不改五官/不美颜”写死，并在出图后核对人是否还是同一个人（见自检）。

### 封面出图法：选帧 + GPT Image 2 叠排（推荐流程）

适用：单组 / 单作品要一张精修封面，且核心图已定稿、不能改人。

1. **选帧**（依 `content-publish-sop` 第二步）：从原图里挑信息密度最高 / 情绪张力最强、缩略图 3 秒能独立读出”这是什么”的一帧做底图；另留 1 张备选。
2. **写 prompt**（不写死成图，只交付指令）。结构固定四段：
   - 开头锁定：”以我上传的这张图为封面底图，保持人物面部 / 姿态 / 关键道具 / 背景全部不变，不要重绘人物、不要美颜、不要改五官。只做以下竖版封面排版叠加，裁成 3:4。”
   - 排版层：标题（题材对应字体/颜色，见上文”封标设计语言随题材切换”）放进暗部/留白、避开五官高光；+ 细线 / 印章 / 柔暗角等该题材的点缀。
   - 留白约束：”除作品名（与印章）外不要任何文字 / 作者名 / 栏目名 / 期号 / 英文 / logo / 边框 / 贴纸。”
   - 气质收尾：”要像电影感写真封面，不要像海报模板或 PPT。”
3. **比例**：主出 3:4（竖屏）；需要横屏再让其改 16:9（人物居一侧、另一侧留白放标题）。
4. **回存与重抽**：成图存项目 `封面/`，命名标用途/比例；**GPT 偶尔写崩中文字形或多加英文，崩了就重抽**或再次强调”只要这几个字”。
5. 范例：`48_即兴抽卡` 合集 11 组「绯雪」`封面prompt_gpt-image-2.md`。**注意设计语言要随选定的那一帧的明暗走，而非随题材名走**：同是"武侠"组，选高调薄雾底（`11_06`）就用高调·冷调语言（深冷墨宋体 + **白柔光晕托字** + 绯红细线 + 朱砂印、**不压暗角**）；选暗调暖底（`11_01`）才用暗调戏剧语言（骨白宋体 + 朱红线 + 朱砂印 + 柔暗角）。**底图是高调近白还是暗调，决定字用深色还是浅色、压不压暗角**——选错会糊。

## 交付格式

交付时给：

- 最终图片路径
- 简短说明：采用了什么比例、保留了什么、删掉了什么
- 如有必要，给一个平台适配建议

不要长篇解释设计理论。用户需要的是可发布的图。
