# AIGC Poster Layout

## 适用场景

当用户要把一张已经满意的 AIGC 作品做成宣传海报、视频封面、分享图、发布预告图时使用本 skill。重点是保护原图的视觉资产，而不是重新生成一张“更像设计模板”的图。

典型触发语：

- “把这张图做成 B 站封面 / 宣传海报”
- “做一张给粉丝看的分享图”
- “不要改人，只做排版”
- “这张是核心锚点，帮我做发布图”
- “不要 PPT 感，不要流程图”

## 核心原则

1. **先保人，再做设计**：角色 IP 图里，“还是不是她/他”高于海报感。
2. **原图比例优先**：不要默认改成 16:9。先判断原图的比例是否就是气质来源。
3. **少字即传播**：粉丝向宣传图只保留角色名、情绪钩子、极小系列信息。
4. **利用原图暗部和留白**：文字应该长在图里，不要做硬切信息栏。
5. **手工排版优先**：核心锚点图不轻易交给图像模型重绘主体。

## 工作流程

### 1. 先做用途判断

明确这张图属于哪一种：

- 粉丝宣传图：追求第一眼情绪和角色吸引力。
- 平台封面图：追求缩略图可读性。
- 项目说明图：允许流程和信息密度。
- 知识复盘图：允许结构化文字，但不适合作为粉丝封面。

如果用户说“给粉丝看”“发 B 站”“视频宣传”，默认按粉丝宣传图处理。

### 2. 评估原图优势

观察原图优势来自哪里：

- 近景脸部：优先保留原比例，不要先横裁。
- 姿态和身体线条：保留全身或半身空间。
- 场景叙事：可以考虑横版扩展。
- 留白天然存在：文字放入留白。
- 没有留白但有暗部：用轻渐变压出文字空间。

不要机械套用平台比例。平台适配版可以作为第二输出，不应覆盖主视觉版。

### 3. 定文字层级

粉丝宣传图**最多两层**（默认更少）：

```text
第一层：角色名 / 作品名
第二层：一句情绪钩子（≤2 行；磁场感题材宁可只留作品名）
```

**封面不挂运营信息**——以下一律不上封面，只写进 README / 简介：

- 作者 ID / 昵称（如「跳蛛先生」）
- 系列名 / 栏目名（如「即兴抽卡」）
- 期号 / 编号 / 日期 / 署名

> 用户明确反馈（2026-06-23）：封面上不要出现作者名、栏目名这类信息；文字越少越好，编辑/时尚向尤其要克制（栖白案例：报头 + 两行封面语 + 作品名即够）。

避免这些词：制作中、工作进度、流程、已完成、归档、版本、prompt、moodboard、face locked。这些词可以写进复盘，不要写在粉丝封面上。

### 4. 排版规则

- 文字避开眼睛、鼻梁、嘴唇、脸部高光。
- 只在自然暗部、头发暗部、背景虚化、前景花影里放字。
- 不做硬分屏，不做独立黑色信息栏。
- 不使用图标、箭头、流程节点、贴纸、UI 卡片。
- 标题可以竖排，但要贴合画面气质；不要像菜单或公文标题。
- **封标设计语言要随题材切换，别一套模板套到底**：
  - 古风 / 国风：宋体（或思源宋体）竖排 + 楷体诗句 + 暖金细线；色走米白 / 暖金 / 灰绿。
  - 暗调戏剧 / 哥特：宋体竖排 + **朱红**细线 / 朱砂印章 + 角落柔暗护主体；弃暖金，用血红 + 大留白造张力。
  - 时尚 / 编辑 / 都市：仿杂志报头——**Playfair/Didot 系高反差衬线报头置顶**（可半隐于发丝），衬线报头 + 无衬线封面语（Didone 小字会断笔）。
  - 二次元 / 仙侠 cosplay / 高调薄雾：冷调空灵——冷墨蓝宋体竖排 + **冰蓝**细线（呼应蓝瞳/刃光），落进薄雾留白。
  - 脚手架（cover-fit / 柔投影 / 2x 超采样）可复用，只换文本层。
- **高调/近纯白底要反过来用深字**：浅底上浅字/暖金会糊，改用冷墨蓝/深字，并在字下垫一层**白色柔光晕**（由文字 alpha 染白 → 高斯模糊 → 垫底）+ 极淡冷投影做分离；别盲目套深底封面的浅色标题。
- 字体落坑：辅助黑体优先 `msyh.ttc`（微软雅黑）；**`HYZhongHeiTi-197` 缺大量中文字形会出豆腐块**。Windows 无 Didot，Playfair Display 是 OFL 可 `curl` 直拉，随项目存 `_fonts/`。
- 颜色：默认米白；按题材加单一强调色（暖金 / 朱红 / 暖陶土），避免亮黄、霓虹、纯白大面积刺眼。

### 5. 输出自检

交付前必须看图自检：

- 缩小后还能第一眼看到脸吗？
- 文字有没有遮住最美的五官和光影？
- 是否像海报，而不是 PPT 或汇报页？
- 是否仍然是同一个角色？
- 字数是否还能再少？
- 成品文件名是否明确标注 FINAL 或用途？

## 工具选择

优先级：

1. 手工排版：适合已定稿核心图、角色锚点图。
2. 代码排版：适合批量封面、重复模板、确定尺寸。
3. gpt-image / 图像生成：适合探索海报方向，不适合锁定人脸的最终版。
4. Midjourney：适合重新出主题图，不适合直接给定稿图加字。

当用户要求“不要改人”时，避免调用图像生成重绘主体。

## 交付格式

交付时给：

- 最终图片路径
- 简短说明：采用了什么比例、保留了什么、删掉了什么
- 如有必要，给一个平台适配建议

不要长篇解释设计理论。用户需要的是可发布的图。
