# 给你的 Windows 小工具/小游戏做「国内直连、零维护」的下载入口

> 适用:你用 vibe coding 做了个 Windows 小程序(.exe),想放到自己的小站上,让国内朋友**点一下就能下**,而且以后发新版**不用每次手动搬包**。

## 先说结论(一句话)

把安装包**托管在你自己的海外(香港)服务器**上,让**自动化流水线**在每次部署时**从 GitHub 的发布页自动抓最新安装包**放上去,网站按钮指向服务器上一个**固定名字**的下载地址。这样:国内直连、速度稳;安装包不进代码仓库;发新版自动同步,你什么都不用改。

> 名词小抄(后面会用到):
> - **GitHub Release(发布页)**:GitHub 上给一个版本挂安装包的地方,每个安装包有一个公开下载链接。
> - **CI / 流水线(GitHub Actions)**:你 `git push` 后自动跑的脚本,能帮你构建网站、把文件传到服务器。
> - **rsync**:把一批文件同步到远程服务器的命令。
> - **dist**:网站构建出来的、准备上线的成品文件夹。

## 为什么不用那些「看起来更简单」的办法

- ❌ **把 .exe 塞进代码仓库**:每发一版就多存一份上百 MB 的大文件,仓库越来越臃肿、克隆越来越慢。
- ❌ **只甩 GitHub 的下载链接**:GitHub 在国内时快时慢,朋友经常下到一半断,体验差。
- ❌ **传到网盘(夸克/百度等)**:每发一版都要手动重新上传替换,朋友还得「转存」才能下,麻烦。
- ❌ **放到 GitHub Pages**:它对单个文件有 100MB 上限,稍大的安装包直接放不上去。
- ✅ **正解**:你网站本来就部署在一台海外服务器上(国内能直连),那就让流水线顺手把安装包也放上去——和网站同一台机器、同样国内直连。

## 怎么做(照着抄)

前提:你已经有 ① 一台海外(香港)轻量服务器在跑你的网站;② 安装包发布在 GitHub Release 上(公开仓库)。

**第 1 步:让流水线在部署时抓安装包。** 在你的部署脚本(GitHub Actions 的 yml)里,「构建网站」和「同步到服务器」中间,加一段:

```yaml
- name: 抓最新安装包放进网站成品里
  env:
    GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: |
    mkdir -p dist/app/download
    # 从你的发布页找到 .exe 的下载地址
    url=$(curl -sL -H "Authorization: Bearer $GH_TOKEN" \
      https://api.github.com/repos/你的用户名/你的仓库/releases/latest \
      | grep -o '"browser_download_url": *"[^"]*\.exe"' | head -1 \
      | sed -E 's/.*"(https[^"]+)".*/\1/')
    test -n "$url"                 # 没找到就让流程报错,别静默继续
    curl -fL "$url" -o dist/app/download/App-Windows.exe
```

**第 2 步:网站下载按钮指向固定地址(用完整网址)。**

```html
<a href="https://你的域名/app/download/App-Windows.exe">下载 Windows 版</a>
```

**第 3 步:照常把 `dist` 同步到服务器(你原本的 rsync 那步不用改)。**

完成。以后你只要在 GitHub 上**发一个新版本**,下次网站部署时安装包就**自动换成最新的**,网站和按钮一个字都不用改。

## 四个会咬人的坑(务必注意)

1. **文件名固定、不带版本号**(用 `App-Windows.exe`,别用 `App-v1.2.3.exe`)。这样网站链接永远不变,发新版你不用改网页。版本号写在网页文字里就行。
2. **抓包这步每次部署都要跑**。很多人的同步命令带「删掉服务器上多余文件」的开关(`--delete`);如果某次没抓包,成品文件夹里没有它,同步时就会**把服务器上的旧安装包删掉**,下载直接 404。所以别用「只在某些情况下才抓」的条件跳过它。
3. **抓失败要让整个流程报错**(上面 `curl -fL` 的 `-f` 和 `test -n` 就是干这个的)。宁可这次部署失败、服务器留着旧包,也不要把一个空文件传上去再被删除命令清空。
4. **只在「同步到服务器」那条流程里抓**,别放进发布到 GitHub Pages 的那条(Pages 单文件 100MB 上限,大包会失败)。

## 一个省事的小技巧

如果只是改了个图标这种小修、而且这个版本**刚发布、几乎没人下载过**:不必升版本号,直接重新构建,把新包覆盖上传到原来那个 Release(`gh release upload 版本号 文件 --clobber`),再手动触发一次部署即可。**但注意**:一旦这个版本已经有人下载了,就别再偷偷改它的内容了——老老实实发个新版本号,免得有人手里的「同一版本」其实不一样。
