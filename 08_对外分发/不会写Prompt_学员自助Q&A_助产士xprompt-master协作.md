# 不会写 Prompt？两个 Skill 帮你从"脑子里有画面"到"手里有提示词"（学员自助 Q&A）

> **这篇文章解决什么问题**：你有一个模糊的创作想法，但不知道怎样把它变成 AI 工具能读懂的专业提示词（Prompt）。
> **我们的方案**：让两个 AI 技能（Skill）协作——**助产士（Maieutic）帮你想清楚你到底要什么**，**Prompt Master 帮你把想清楚的东西翻译成专业提示词**。
> **你能收获什么**：读完这篇，你能自己跑完"模糊想法 → 清晰需求 → 可粘贴的成品提示词"全流程，不用再来问助教。

---

## 先读这篇能帮你少走多少弯路

在正式回答之前，先对齐一个认知：**写不好 Prompt 通常不是"不会写字"，而是"没想清楚"**。

大部分学员卡住的地方是：
- 脑子里有画面，但说不出来
- 说出来了，但 AI 出的东西不是自己要的
- 知道 AI 出的不好，但不知道改哪里

这三个问题分别对应：**需求没收敛 → 需求没翻译 → 反馈没到位**。我们的两个 Skill 就是专门解决前两个的。

---

## Q1：什么是 Skill？为什么能帮我写 Prompt？

**Skill 就是一本给 AI 的"岗位手册"**。

你每次跟 AI 对话，它其实都"失忆"——不记得你喜欢什么格式、不知道你反复讲的规则、也不记得上次踩过坑。Skill 就是你提前写好的手册：遇到什么情况、按什么规则处理、最后交付什么格式。

具体到写 Prompt 这件事，我们准备了两本手册：

| Skill | 它干什么 | 类比 |
|---|---|---|
| **助产士（Maieutic）** | 用苏格拉底式提问帮你**想清楚**你到底要什么 | 像一个有耐心的策划，不替你决定，但帮你把模糊想法收敛成清晰需求 |
| **Prompt Master** | 把收敛好的需求**翻译**成某个 AI 工具能读懂的专业提示词 | 像一个懂各地方言的翻译，知道 Midjourney 怎么说、Claude Code 怎么说、Cursor 怎么说 |

**为什么不让一个 Skill 全干了？** 因为"想清楚"和"翻译"是两种完全不同的思维活动。合在一起容易两边都做不好——要么替你决定了（你不满意），要么翻译时漏掉关键信息（AI 听不懂）。拆开才能各司其职。

---

## Q2：Skill 怎么工作？我该先跟谁说？

整个流程分两步，**严格按顺序**：

```
你的模糊想法
     │
     ▼
┌─────────────────────────────┐
│  第一步：助产士（Maieutic）  │
│  帮你收敛需求               │
│  输出：一段清晰的需求描述    │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  第二步：Prompt Master       │
│  把需求翻译成专业提示词       │
│  输出：可直接粘贴的成品 Prompt │
└─────────────────────────────┘
```

**关键原则：不要跳过第一步。**

很多学员上来就说"帮我写一个 Midjourney 的 prompt"，然后直接让 Prompt Master 出词。这样出的东西往往不对——因为 Prompt Master 会基于你模糊的描述做猜测，猜错的方向越精致越浪费时间。

正确做法：**先让助产士问你几个问题，回答完再让 Prompt Master 写**。

## Q3：我完全不想装东西，能直接用吗？

**能。我们有三种方式，从"零安装"到"手动安装"，你挑适合自己的。**

### 方式一：直接上传 Skill 文件给 AI（推荐，零安装，通用）

这是最稳的方式，**不需要安装任何东西，Claude / GPT / 国内模型都能用**。原理：直接把"手册"文件发给 AI，AI 读完就知道怎么干。

**Skill 文件在知识库中的位置**：

| Skill | 知识库路径 | 说明 |
| --- | --- | --- |
| 助产士（Maieutic） | `07_skill存档/maieutic-skill/SKILL.md` | 含苏格拉底式提问的四模式判断规则 |
| 助产士 — 系统提示词 | `07_skill存档/maieutic-skill/prompts/system_prompt.md` | 身份、灯塔原则、禁止事项 |
| 助产士 — 模式分类器 | `07_skill存档/maieutic-skill/prompts/mode_classifier.md` | Knowledge / Exploration / Reflection / Creation 模式判定 |
| 助产士 — 输出模板 | `07_skill存档/maieutic-skill/prompts/reflection_output.md` | Insight / Beacon 标准输出模板 |
| Prompt Master | `07_skill存档/prompt-master/SKILL.md` | 全工具路由的 prompt 工程规则 |
| Prompt Master — 模板库 | `07_skill存档/prompt-master/references/templates.md` | 13 套场景模板（RTF / CO-STAR / Midjourney / Cursor / Claude Code 等） |
| Prompt Master — 模式库 | `07_skill存档/prompt-master/references/patterns.md` | 35 种 Prompt 反模式诊断参考 |

**你只需要做一件事**：每次开始新对话时，把对应 Skill 文件的内容复制粘贴发给 AI（或在对话里上传文件），加一句：

```
请按照这份手册的规则，先用助产士帮我想清楚需求，需求明确后再用 Prompt Master 写提示词。
```

> **注意**：AI 每次开新对话都会"失忆"，所以每次新对话都要重新发一次文件。建议把文件保存到本地，随时粘贴。如果用的是 Claude Code，也可以把文件内容存成项目级的 CLAUDE.md 或 AGENTS.md，每次启动自动加载。

### 方式二：手动安装（适合 Claude Code / Codex 用户）

如果你用的是 Claude Code 或 Codex，可以把两个 Skill 装到对应目录：

**Claude Code**：
```
%USERPROFILE%\.claude\skills\maieutic-skill\SKILL.md
%USERPROFILE%\.claude\skills\prompt-master\SKILL.md
```

**Codex**：
```
%USERPROFILE%\.codex\skills\maieutic-skill\SKILL.md
%USERPROFILE%\.codex\skills\prompt-master\SKILL.md
```

安装后**完全退出并重启客户端**，Skill 会在新对话里自动生效。

> ⚠️ Prompt Master 还有两个附属文件（`references/templates.md` 和 `references/patterns.md`），如果只装了 `SKILL.md`，部分模板会缺失。建议把整个 `prompt-master` 文件夹一起复制过去。

### 方式三：直接发文件给 AI（最轻量）

如果你只是临时用一下，不想做任何安装：

1. 把 `maieutic-skill/SKILL.md` 和 `prompt-master/SKILL.md` 这两个文件的内容复制下来
2. 发给 AI，说："请按照这份手册的规则帮我……"

这种方式每次都要发一次文件，但胜在零配置。

## Q4：我第一步具体该说什么？

不管你用哪种方式激活 Skill，**第一轮对话建议这样说**：

```
我想做 <描述你的模糊想法>，
但不知道怎么把它写成 <目标工具名> 能读懂的提示词。
请先用助产士帮我想清楚需求，需求明确后再用 Prompt Master 写提示词。
```

**举个具体例子**：

```
我想做一张高端雪山单板广告图，要有电影感、速度感、很贵的感觉，
但不知道怎么把它写成 Midjourney 能读懂的提示词。
请先用助产士帮我想清楚需求，需求明确后再用 Prompt Master 写提示词。
```

这样 AI 就知道：先启动助产士模式，不要急着出词。

## Q5：助产士会问我什么？我该怎么回答？

助产士会根据你的问题，自动判断你处于哪种模式，然后问你最关键的 1-2 个问题：

| 模式 | 你会有的感受 | 助产士会问什么 |
|---|---|---|
| **知识模式** | "什么是……""X 怎么工作" | 一般不问，直接回答 |
| **探索模式** | "我想学/做……但不知道从哪开始" | 问你的动机和目标 |
| **反思模式** | "我很迷茫/卡住了/不知道自己要什么" | 先理解你的状态，再澄清 |
| **创作模式** | "我想做一个……" | 帮你结构化问题、暴露假设 |

**回答原则**：
- **不用一次说完**：助产士一次只问 1-2 个关键问题，你回答就行，不用写小作文
- **想到什么说什么**：不用组织语言，口语化完全没问题
- **不知道就说不知道**：助产士会基于"不知道"继续收敛，这本身就是有效信息

## Q6：助产士问完了，接下来怎么切到 Prompt Master？

当助产士帮你收敛出一段清晰的需求描述后（通常 3-5 轮对话），你可以说：

```
需求已经清楚了，现在请用 Prompt Master 帮我写提示词。
目标工具是 <Midjourney / Claude Code / Cursor / ……>。
```

**Prompt Master 会做什么**：
1. 确认目标工具（如果你没说，它会问）
2. 从你收敛好的需求里提取 9 个维度（任务、格式、约束、输入、上下文、受众、成功标准等）
3. 按目标工具的语法生成一条可粘贴的成品提示词
4. 给出简短说明：优化了哪一点、为什么这样写

**Prompt Master 的输出长这样**：

````
Role: 你是一位高端运动品牌广告摄影师...
Task: 创作一张...画面
Format: 单张 16:9 横版,...

🎯 Target: Midjourney, 💡 把"很贵的感觉"转译为具体的光影与构图参数
````

直接复制提示词块，粘贴到目标工具就行。

## Q7：Prompt Master 出的提示词不好怎么办？

**先判断是哪个环节的问题**：

| 现象 | 问题出在哪 | 怎么修 |
|---|---|---|
| AI 出的东西完全不是你要的 | 需求没收敛够（助产士阶段） | 回到助产士，补充"我要的不是XXX" |
| 方向对但细节不对 | 翻译不到位（Prompt Master 阶段） | 直接告诉 Prompt Master "把XX改成YY" |
| 出了几张，不知道选哪张 | 缺成功标准 | 跟助产士说"帮我定义什么叫'好的'" |
| 提示词太长/太短 | 格式约束没说清 | 告诉 Prompt Master "控制在 50 词以内" |

**反馈模板**（直接复制改）：

```
这条提示词方向对了，但 <具体哪里不对>。
请保留 <具体哪部分>，把 <哪部分> 改成 <你想要的描述>。
```

## Q8：我用的不是 Claude/GPT，是国内模型怎么办？

如果你用的是 DeepSeek、通义、豆包、Kimi 等国内模型，**助产士部分有专门的适配包**：`maieutic-deepseek-adapter`。

这个适配包把助产士的核心行为（四模式判断、Insight/Beacon 输出规则）翻译成了国内模型能稳定执行的 prompt 包。

**使用方式**：
- DeepSeek：把 `prompts/system_prompt.md` 放入系统提示词，其余作为开发者说明
- 通义/豆包/Kimi：作为 Prompt Pack 手动拼接

> ⚠️ Prompt Master 目前主要面向 Claude / GPT / Midjourney / Cursor 等工具。如果你的目标工具是国内模型，Prompt Master 出的提示词可能需要微调，但收敛需求（助产士那一步）是完全通用的。

## Q9：常见坑位怎么避？

### 坑一：跳过助产士，直接让 AI 写 Prompt

**症状**：AI 出了一段很"专业"的提示词，但出的东西不是你要的。
**原因**：AI 基于模糊描述做了错误猜测，越精致越偏离。
**避法**：先跑助产士，让需求收敛成一段你自己能念出来的描述，再进 Prompt Master。

### 坑二：一次塞太多需求

**症状**：你说"既要电影感又要卡通感，既要快又要慢"，AI 出的四不像。
**原因**：一个提示词只能服务一个核心意图。
**避法**：一次只做一个核心意图。想做的多？拆成多条提示词，每条一个目标。

### 坑三：提示词里全是形容词，没有动词和名词

**症状**："高端的、大气的、有质感的、电影感的……"
**原因**：形容词 AI 不知道怎么执行，它需要具体的名词（什么物体）和动词（做什么动作）。
**避法**：把形容词翻译成可执行描述。"高端" → "浅景深、冷色调、单一主体居中"；"电影感" → "2.35:1 宽银幕、低调照明、胶片颗粒"。

### 坑四：忘了告诉 AI 目标工具

**症状**：提示词写得很漂亮，但粘贴到 Midjourney 里参数不生效。
**原因**：不同工具语法不同。Midjourney 用 `--ar 16:9`，DALL-E 用自然语言描述比例。
**避法**：一定告诉 Prompt Master 你的目标工具是什么，它会按对应语法生成。

### 坑五：每次新对话都要重新发开场白

**症状**：上次用得好好的，今天再问"它怎么不认识助产士了"。
**原因**：AI 每次新对话都会"失忆"，Skill 不会自动跨对话记住。
**避法**：把开场白存成快捷短语，每次新对话粘贴一次，几秒钟的事。

## Q10：我想系统学会写 Prompt，下一步学什么？

跑通上面的流程后，如果你想更进一步：

1. **读一条好提示词是怎么来的**：找一篇你喜欢的成品 Prompt，用 Prompt Master 的"反编译模式"（Prompt Decompiler）拆解它——它会自动分析这条提示词的结构、意图和可迁移模式
2. **建立自己的提示词模板库**：每次跑通一条，把它存下来，标注"什么场景下用、效果如何"，慢慢你就有了自己的武器库
3. **学 Prompt 的理论基础**：读 [[SKILL入门完全指南]] 里"判断一个 Skill 好不好"那节，把标准反过来用，就是判断一条提示词好不好的标准
