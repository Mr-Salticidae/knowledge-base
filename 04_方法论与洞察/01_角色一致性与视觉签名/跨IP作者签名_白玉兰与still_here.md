# 跨 IP 作者签名 · 白玉兰与 still here

> 入档：2026-05-12（D2 晚间）
> 触发：「拾色」立项时发现两条跨 IP 共享元素
> 性质：账号级长期资产 · 视觉签名 + 听觉签名

---

## 一句话

**多 IP 账号需要"共享签名"**——观众看到任意一张图 / 听到任意一段歌，能立刻识别"这是同一个作者"。

跳蛛先生账号已经有两条跨 IP 签名：

1. **视觉签名：白玉兰**（檐下 + 拾色共享 / R-07 因无人形不挂）
2. **听觉签名：still here**（R-07 + 拾色副歌共享）

---

## 如何使用

未来开新 IP 时，**先问**：

1. 这个新 IP 能挂"白玉兰"元素吗？挂在哪里？（位置可不同，元素本身要在）
2. 这个新 IP 的副歌能不能写一句以 "still ___" 开头的英文 hook？

如果两个都答"能" → 强签名 IP（自动接入账号美学体系）
如果答"不能" → 需要明确陈述"为什么这个 IP 不挂签名"（如 R-07 因无人形）

## 白玉兰跨 IP 签名

### 来源

「檐下」V3 角色卡（2026-05-01）的实测发现：

> 玉兰是 AI 在 personalize 训练里"投票"选出的视觉签名，从此锁定为「檐下」IP 视觉锚点。

### 跨 IP 复用

| IP | 玉兰位置 | 形态 | 与 IP 内核的呼应 |
|---|---|---|---|
| **檐下**（古代 · 雪 · 木檐）| 头顶 | 一朵白色玉兰花簪 | 古典少女发饰 |
| **拾色**（民国 · 雨 · 旗袍）| **耳后/耳侧** | 小白玉兰 + art deco 银 hairpin 组合 | 民国摩登发饰，玉兰若隐若现 |
| R-07（机器人 · 末日）| ❌ 不挂 | — | 无人形，挂签名违背 IP 内核 |
| 14_我的一天（末世孤存者）| ❌ 不挂 | — | 同上，背对镜头无头部展示 |

### 设计原则

- **元素不变**（白玉兰）
- **位置可变**（头顶 / 耳后 / 衣襟）
- **形态可变**（大花 / 小花 / 含苞）
- **共生道具可变**（hanfu 立领 / 旗袍 hairpin / 现代衣领）

观众看到任意一张图里有白玉兰 + 这种调性 → **一眼认出"跳蛛先生作品"**。

## "still here" 跨 IP 副歌签名

### 来源

R-07 Suno V-A_001 副歌：

```
still here, still near
something soft, something dear
oooh, oooh...
still here, still near
```

「拾色」Suno V-S_001 副歌（v5.5 自动作词，自发对位）：

```
Still here, still here
Still kept, still kept
Still folded, still folded
Still unread
```

### 共享公式

`still + [形容词/状态词] × 重复回响 + 短英文 hook`

### 未来 IP3/IP4 延续

| IP | 副歌 still 句式（候选）|
|---|---|
| 假想 IP3（晨光线）| `still bright, still warm` |
| 假想 IP4（深夜线）| `still awake, still listening` |
| 假想 IP5（雪线）| `still falling, still white` |

只要保持 "still + 形容词" 的英文 hook + Mandarin 主歌 + 慢节奏，**听众听 5 秒就能识别"这是跳蛛先生作品系列"**。

## 跨 IP 签名 vs 单 IP 签名

**单 IP 签名**（如「檐下」的"古典少女 + 头顶玉兰"）= 锁这一个 IP

**跨 IP 签名**（白玉兰 + still here）= 锁整个**账号美学**

跨 IP 签名比单 IP 签名更高级：
- 抗 IP 老化（一个 IP 消亡，签名仍在新 IP 里延续）
- 跨平台识别（B 站 + 抖音 + 网易云 + 小红书都成立）
- 系列化扩展性（新 IP 不需要从零建立心智）
