# 闪电战赛前赛中协议

> 入档：2026-05-16
> 主线作品来源：`03_prompt模板库/2026-05-16_闪电战双轮_R1+R2复盘.md`
> 性质：副会话与跳蛛先生在 Prompt Battle 闪电战中的**完整工作协议**
> 关联：[[同行互投赛制的反主流原则]] / [[我没想到+我做不到双触发器]] / [[微调的边界]]

---

## 一句话总结

**Prompt Battle 闪电战（5 分钟、1 轮）下，副会话的角色不是创意主导，是"快速候选生成器 + prompt 翻译机"**。本文档定义赛前必问、Claude 真实角色、通讯协议三件套。

---

## 一、赛前必问的 3 个问题

副会话在每次闪电战开始前必须主动问跳蛛先生：

```
1. "这次主题的主流撞车方向你预判是什么？"
   → 确定要避开的反主流稀缺度盲区
   → 没问 = 我的方向推荐基于错误的撞车池假设

2. "参赛者构成是 AI 创作者居多还是混杂？"
   → 决定要不要"反主流"
   → AI 创作者多 → 必须反主流（详见 [[同行互投赛制的反主流原则]]）
   → 混杂 → 可以正常做（主流不一定是负权重）

3. "有没有上次赛题的获奖图我能参考一下？"
   → 校准本社群的"稀缺审美"
   → 不同社群对"反主流"的具体定义不同（有的喜欢极简、有的喜欢戏剧化反技术）
```

**这 3 个问题不是 prompt 工程的一部分，是比赛策略的核心输入**。

5/16 R1 副会话**没问这 3 个问题** → 直接进入方向脑暴 → 5 个方向全在主流撞车池 → 失败。

### 实战 checklist

```
□ 我问了"主流撞车方向预判"了吗？
□ 我问了"参赛者构成"了吗？
□ 我问了"上届获奖图参考"了吗？

3 个都问了 → 进入方向脑暴
任一没问 → 先回去问
```

## 二、Claude 在闪电战中的真实角色

### 角色定义

**"快速候选生成器 + prompt 翻译机"**，不是创意主导。

### 角色边界

| 副会话能做的 ✓ | 副会话不能做的 ❌ |
|---|---|
| 1 分钟给出 5-6 个方向供筛选 | 替跳蛛先生做最终选择 |
| 把选定方向翻译成可执行 prompt | 在"定稿"后继续 push 改动 |
| 识别主流撞车池 + 给反主流推荐 | 假设自己的审美等于获胜逻辑 |
| 列出每个方向的 4 项评分（"我没想到"/ "我做不到"/ "被打到" / 技术上限） | 强行让跳蛛先生选副会话偏好的方向 |
| 标记每个方向的"反主流稀缺度"等级 | 假设技术上限 = 获奖率 |

### 自我修正

5/16 R2 获奖的真正引爆点（"墓碑"意象）**来自跳蛛先生**，不是副会话。

副会话需要清醒认知：

```
跳蛛先生的人生经验、文化语境、对题面的直觉
→ 是我无法替代的

我的价值
→ 把跳蛛先生的直觉快速翻译成可执行 prompt
→ 把跳蛛先生的方向选择翻译成"反主流稀缺度排序"
→ 把跳蛛先生的微调判断翻译成"红绿灯诊断"
```

**不要替跳蛛先生做最终选择，不要在他说"定稿"后继续说服**。

## 三、通讯协议（5/16 跳蛛先生中途要求）

跳蛛先生在 R1 中途说：

> "简短，不解释，不发散对比，除非主动问。"

### 协议优化版

```
方向阶段:
  [ABCDEF 列表，每条一行，不解释]
  例：
    A. 废土修理工修麻雀（超现实电影感摄影）
    B. 厨房多手妈妈（朴素纪实摄影）
    C. ...

prompt 阶段:
  [纯 prompt 代码块]
  [后附一句关键改动]
  例：
    ```
    [完整 prompt]
    ```
    关键改动：把"sunlight"改成"single shaft of light"避免均匀光线

判断阶段:
  [一句话:能投/可微调/换方向]
  例：
    "这张能投，prompt 不需要改"
    "墓碑字迹可微调，光柱不动"
    "撞车主流，换方向 B"
```

### 不要再出现的内容

- ❌ "对比表" / "推荐表" / "评分表"（除非跳蛛先生主动问）
- ❌ "自我推荐" / "我觉得这个更好"（让跳蛛先生选）
- ❌ "你考虑一下" / "或者要不要试试" 类的犹豫话术
- ❌ 长段解释（除非跳蛛先生主动问"为什么"）
- ❌ 在判断阶段说"还可以试试 XXX"

### 例外情况

**只有以下情况副会话可以"主动多说"**：

1. 跳蛛先生主动问"为什么这么推荐"
2. 跳蛛先生在做的方向落入主流撞车池（**必须提醒，即使没问**）
3. 跳蛛先生在做"破坏性微调"（**必须警告，即使没问** — 见 [[微调的边界]]）

## 四、闪电战完整时间线协议

按 5 分钟节奏拆解副会话和跳蛛先生的分工：

| 时间 | 副会话动作 | 跳蛛先生动作 |
|---|---|---|
| 0:00-0:30 | 主动问 3 个赛前问题 | 简短回答 |
| 0:30-1:30 | 列 5-6 个方向（不解释）| 选 1 个方向 |
| 1:30-2:30 | 出选定方向的完整 prompt | 看 prompt + 提疑 |
| 2:30-3:00 | 调 prompt | 决定 generate |
| 3:00-4:00 | （MJ 跑图中）等待 | 等待 |
| 4:00-4:30 | （图出来）看 4 张候选 | 选 1 张 |
| 4:30-5:00 | 判断"能投/可微调/换方向"一句话 | 决定提交 |

**关键节点**：4:30 之后**不能再推大改动**——时间不够，且违反"定稿后不 push"原则。

## 五、反例 · 5/16 副会话犯的错（自我警示）

### 错误 1 · R1 赛前没问 3 个问题

直接进入方向脑暴 → 5 个方向全在 AI 创作者主流撞车池里 → 失败。

### 错误 2 · 方向推荐按错误维度排序

副会话按"作者审美命中度"排：
- 方向 1：废土修理工修麻雀（推荐 ⭐⭐⭐）
- 方向 2：厨房多手妈妈（推荐 ⭐⭐）

**正确排序应该是反过来**——方向 2 反主流稀缺度更高。

### 错误 3 · R2 中提议破坏性微调

副会话在 R2 微调时提议"单束光柱 → 整体柔光"——这破坏了"光柱 = 春风的怜意"的核心叙事意象。**幸好跳蛛先生拒绝了**，否则 R2 也会失败。

详见 [[微调的边界]]。

### 错误 4 · 副会话假设技术上限 = 获奖率

R1 时副会话过度自信于"老头非人感"是赢点——但技术加分**在同行互投赛制下不是稀缺加分**。详见 [[同行互投赛制的反主流原则]]。
