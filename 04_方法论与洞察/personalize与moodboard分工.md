# personalize 与 moodboard 的分工

> 首次成型:2026-05-01(02 教程账号级锚点章节)
> 升级:2026-05-01(《众声里的留白》创作时实测)
> 入档:2026-05-02
> 主线作品来源:`02_角色一致性锚点/02_角色卡.md` + `临时创作/2026-05-02/01_众声里的留白.md`

---

## 一句话总结

**personalize 是审美底盘,moodboard 是题材绑定**——personalize 默认常驻,moodboard 按题材开关。

---

## 如何使用

| 你要做什么 | 怎么开关 |
|---|---|
| 跑「檐下」相关人像 | personalize 开 + moodboard 开 |
| 跑非人像题材(抽象 / 风景 / 动物 / 概念) | personalize 开 + moodboard **关** |
| 想看"完全没有我审美干预"的 base 输出 | personalize **关** + moodboard 关(对照实验用) |

---

## 关键差异表

| 维度 | personalize | moodboard |
|---|---|---|
| **性质** | 静态、被动训练 | 题材绑定的参考池 |
| **触发** | 200+ 张图打分 | 上传一组样片 |
| **作用范围** | **跨题材**(底盘审美) | **限题材**(只对训练时的题材有效) |
| **类比** | 你的"审美 DNA" | 给 MJ 看一组"我喜欢这种样片" |
| **跨版本兼容** | V7→V8.1 兼容 ✅ | V7→V8.1 兼容 ✅ |
| **遇到非题材怎么办** | 仍生效(它是底盘) | 失效或反向干扰(它是题材绑定) |

---

## 实测验证

### 验证 1 · 人像题材(《檐下》)

- personalize 开 + moodboard 开 → 完美命中跳蛛先生的"清冷古风少女"审美
- personalize 开 + moodboard 关 → 仍是中国脸,但缺少 moodboard 的"质感细节"
- personalize 关 + moodboard 关 → 出"复古东方美人"(西方训练集偏差,见 [[AI甜妹脸vs复古东方美人]])

### 验证 2 · 非人像题材(《众声里的留白》抽象插画)

- personalize 开 + moodboard 开(默认状态)→ moodboard 强行把"古风女性元素"塞进抽象画(灾难)
- personalize 开 + moodboard **关** → 完美的抽象插画 + 仍带跳蛛先生的色彩偏好(底盘生效)

**这次的发现极其关键**:**moodboard 是题材绑定的,它在非自己题材里反而是负担**。

---

## 战略含义

### 1 · 写 prompt 前的开关 checklist

- [ ] 题材是不是 moodboard 训练时的领域?(古风女性 / 古典摄影)
- [ ] 是 → moodboard 开
- [ ] 不是 → moodboard 关
- [ ] personalize 默认始终开(它是底盘,跨题材都生效)

### 2 · 多 moodboard 战略(未来可能)

跳蛛先生现有 1 个 moodboard:"清冷古风"。
如果未来要做多种风格,**可以训练多个 moodboard**:
- moodboard A:清冷古风(现有)
- moodboard B:抽象概念插画
- moodboard C:独立游戏视觉
- ...

每次跑图按题材选 moodboard,personalize 始终在底盘。

### 3 · 跨版本兼容是隐藏宝藏

很多创作者担心"升级 V8.1 会废掉我的训练"。
**实测验证 personalize 和 moodboard 都跨 V7→V8.1 兼容**——
**这是值得专门做一条作品的"安心剂"内容**。

---

## 一个隐藏的内容点

> "moodboard 是题材绑定的,personalize 是底盘审美。"

这句话听起来是技巧贴,但展开说是**一个 IP 战略层面的洞察**:
- **moodboard 锁审美质感**(图像层)
- **personalize 锁审美方向**(认知层)

跳蛛先生的"清冷古风"是 moodboard 的**显性表达**;
他的"实证理性 / 反水文 / 文学气" 是 personalize 训练里**隐性沉淀**的审美方向。

**两者叠加 = 一个有"性格"的账号**,而不是"用了同款工具的账号"。

---

## 顶层金句段

> *"moodboard 是题材绑定的,personalize 是审美底盘。*
> *跑非人像题材(抽象/插画/概念图)时关掉 moodboard,*
> *但 personalize 默认开,因为它是你 200 张图打分训出来的整体审美,*
> *不只对人像生效——它在抽象题材里同样在悄悄'调味'。"*

> *"personalize 锁审美方向,moodboard 锁审美质感。*
> *缺前者,你只是用了同款工具。*
> *缺后者,你的图缺了一层无法言说的细腻。*
> *两者叠加,你的账号才有'性格'。"*

## 升级补丁(2026-05-02 自由探索副会话二 · 胶片人像探索)

> 来源:在跑"清冷东方少女回眸"系列时,加入"清冷古风"moodboard 让产出有质的飞跃 —— 把 prompt v5(西式胶片人像题材)从"OK"推到了"惊艳"。
> 实测样本:`临时创作/2026-05-03_multi_sref探索/胶片人像探索/` 的 5 张"清冷少女回眸"

### 修正一:Moodboard 的贡献维度比"肌理"更广

之前理解 moodboard 是"题材绑定的肌理参考池"。这次发现它**还会贡献**:

- **情绪 register**(清冷气质 vs 暖光私密)
- **姿态 archetype**(把 prompt 模糊句 `head turned toward window light` 解读成"回眸看观者")
- **色调倾向**(冷调微调)

**修正性质**:Moodboard 不只是"肌理参考池",更准确的说法是 ——
**它在 prompt 没强力指定的所有维度上,自动调味**。

### 修正二:"自家题材边界"按主体定义,不按场景题材定义

之前那条"moodboard 在非自家题材里反向干扰"判断,基于 `众声里的留白` 抽象插画 —— 那是**主体不匹配 + 题材不匹配**。

这次场景是 **主体匹配(清冷东方少女)+ 题材不匹配(西式胶片人像)** —— moodboard **没干扰,反而精准贡献了主体维度**。

更准确的规则:

> **Moodboard 的"自家题材"边界主要按主体定义。**
> 主体匹配 → 不冲突的所有 prompt 模糊处都是加成
> 主体不匹配 → 强行套主体导致干扰

### 修正三:Moodboard vs Prompt 的"影响力分配"机制

新增一条心智模型:

> **Prompt 强约束的维度**(环境、构图、服装色、时间) → moodboard 影响小
> **Prompt 模糊或未指定的维度**(气质、姿态、面孔细节、色调) → moodboard 强势介入

实操含义:**你想 moodboard 贡献什么,就在 prompt 里"留白"什么** —— 别把所有维度都写满。这次 prompt 故意保留 `head turned slowly toward the window light` 这种带歧义的描述,给了 moodboard 把它解释成"回眸"的空间 —— 这是这次出爆款的隐藏机制。

### 修正后的"何时该开 moodboard"判断

旧的 checklist:
- 题材是不是 moodboard 训练时的领域?

修正版 checklist:
- **主体**是不是 moodboard 训练时的目标?(不是题材!)
- Prompt 是不是给 moodboard 留出了"留白维度"让它发挥?

两个都是 yes → 开;任一是 no → 关。

## 升级补丁(2026-05-03 · V4 基准图修复 R2 双盲对照实验)

> 来源:V4 基准图修复训练场 R2,因误操作产生的天然双盲对照(同 prompt × personalize 开/关)
> 实测样本:`09_V4基准图修复训练场/04_产出_待跑/R2_chin-lowered-eyes-meeting-lens_带personalize/` 8 张 vs `R2_chin-lowered-eyes-meeting-lens_没带personalize/` 8 张
> 入档当日:跳蛛先生跑 R2 时不慎只开 moodboard 没开 personalize,补跑了一组带 personalize 的——意外得到天然 A/B 对照样本

### 修正四:personalize 是"审美方向锚"——从理论命题升级为实证事实

之前在文档里说"personalize = 审美底盘 / 审美方向"——这是结构推理,不是实证。

R2 的对照实验首次给出硬证据:
- **同一个 prompt**(`chin slightly lowered, eyes meeting the lens through falling snow, wet eyelashes` + 完整 V4 prompt 主体)
- **同一个 moodboard**("清冷古风" 全程开)
- **唯一变量**:personalize 开 vs 关
- 各跑 8 张

**结果**:
- **几何拓扑两组一致**——"头微低 + 看镜头 + 湿润"在两组都被成功触发,prompt 几何描述跨 personalize 状态稳定
- **气质天差地别**:
  - personalize 开 → 8/8 保持"清冷少女":唇色自然、瞳色正常、表情含蓄、年龄感少女
  - personalize 关 → 8/8 退化为"复古东方美人"陷阱:深酒红唇、红/茶色瞳、妩媚成熟、年龄感拉到 25-30

### 含义:三个锚的分工进一步清晰

| 锚 | 管什么 | 不管什么 |
|---|---|---|
| **prompt** | **几何 / 环境 / 构图 / 服饰** | 气质方向 / 质感 |
| **moodboard** | **质感 / 肌理 / 留白维度的调味**(只在共识维度有效) | 几何 / 气质方向 |
| **personalize** | **气质方向**:唇色取向 / 瞳色取向 / 表情底色 / 年龄感 | 几何 / 质感 |

三者**互不替代**——这是 R2 对照实验最干净的产出。

### 这条与 [[AI甜妹脸vs复古东方美人]] 的关系

R2 的"不开 personalize 组"完美复刻了那篇文档里描述的"复古东方美人"陷阱——同样的深酒红唇、同样的红茶色瞳、同样的妩媚成熟感。

**这是同一陷阱的**双盲实验**版本**——比那篇文档的"裸跑就出复古东方美人"多了"开 moodboard 也救不回来"这个增量证据:**moodboard 救不了 personalize 的缺位**(因为它们管的是不同维度)。

### 实操升级:checklist 升级为"强制三开"

之前 checklist 里 personalize 写"默认开"——现在升级为**强制开,任何"清冷少女"题材都不可关**:

- [ ] personalize **强制开**(关 = 必出复古东方美人,moodboard 救不回)
- [ ] moodboard **按主体边界开关**(主体匹配且想留白调味 → 开;主体不匹配 → 关)
- [ ] prompt **几何写实 + 气质留白**(气质交给 personalize,质感交给 moodboard)
