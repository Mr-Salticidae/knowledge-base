# 展台机易失环境自恢复预案 _v1

> 入档:2026-07-09
> 验证状态:⚠️ 首次发现(预案 07-09 上午入仓,当天深夜 H 盘即被清空,恢复脚本当场应验)
> 来源:《镜像自我·人生预演》BW 展台机(网吧式还原环境),见 [[2026-07-09_镜像自我人生预演_36h极限开发复盘]]

## 一句话

**网吧式还原机上,不在 git 仓库里的一切都是易失的——工具盘、node_modules、git 身份、全局配置、凭据随时清零;预案 = 依赖入仓 + 一键恢复脚本,文档里写死的绝对路径必失效。**

## 实测清单(2026-07-09 一夜之间消失的东西)

- `H:\临时\tools\` 整盘:便携 Node、ffmpeg
- `app/node_modules/`(gitignore 内=不受 git 保护)
- `git config user.name/email`(全局配置)
- 结论:**"已随本机备好一份"是幻觉**,只有仓库内的字节和可重放的脚本是真的

## 操作规则

1. **恢复脚本入仓**:每个外部依赖配一个"缺失即下载解压"的幂等脚本(如 `准备构建环境.ps1`:测路径→官源下载→解压→改名),换机器/被清零后一条命令满血;
2. **launch/构建配置指向仓库内路径**(`tools\node-portable\`),不指向任何盘符绝对路径;
3. **npm.cmd 陷阱**:便携 Node 的 npm.cmd 需要 PATH 上有 node——kiosk 启动项直接用 `node.exe node_modules/vite/bin/vite.js` 绕开;
4. **文档里的工具路径当线索不当事实**:按"路径还在吗→不在就按恢复法重装"的姿态写文档(路径+恢复命令成对出现);
5. git 身份恢复:从 `git log` 历史提交里取回 author,配 repo-local 不动全局。

## 反例/边界

- 自己的常用开发机不必如此偏执;本律适用于**展台机/网吧机/共享机/会被系统还原的一切环境**。
- 大体积依赖(百 MB 级安装包)不宜入仓——用"官源 URL + 校验 + 一键脚本"代替字节入库。
