# emoji 字形版本兼容律 _v1

> 入档:2026-07-09
> 验证状态:⚠️ 首次发现
> 来源:蛛网之上「可玩」区《镜像自我·人生预演》卡片图标,见 [[2026-07-09_镜像自我人生预演_36h极限开发复盘]]

---

## 一句话

**对外 UI 里的 emoji 是在赌用户系统字体——Unicode 新版本字形(约 12.0/2019 之后)在旧 Windows 上渲染成方框豆腐;公开页面选老字形,新字形要么自测旧系统要么换 SVG。**

## 案例

游戏卡片用 🪞(镜子,U+1FA9E,**Unicode 13.0/2020**)——主题贴切,但用户 Win10 机上直接方框。换 🎭(戏剧面具,**Unicode 6.0/2010**)即愈,且"人生预演"的戏剧隐喻反而更贴题。

## 操作规则

1. 公开页面的 emoji 优先从 **Unicode 6.0–9.0 老字库**里选(😀🎭🎬🔥⭐🥁🕵️ 级别的"古董"最安全);
2. 想用新字形时,先查收录版本(emojipedia 一查即知),再在**旧 Windows / 旧 Android** 上实测;
3. 兜底思路:关键图标别依赖 emoji,用内联 SVG/图片——emoji 只当装饰,装饰缺字也不该破相;
4. 语义相近时**老字形优先于贴切度**:方框比"差一点贴切"伤害大得多。

## 反例/边界

- 受众全是新设备的场景(iOS 用户群/内部工具)可放宽;
- 与 [[CJK假斜体语言感知律_v1]] 同族:**"我这里显示正常"不等于用户那里正常,字体渲染永远要在目标环境验**。
