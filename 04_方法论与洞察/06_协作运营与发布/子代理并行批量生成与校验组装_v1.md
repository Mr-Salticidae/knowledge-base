# 子代理并行批量生成 + 脚本校验组装

> 入档:2026-06-24
> 来源:把《反诈柜台》案件库从 16 桩扩到 200 桩(184 桩双语结构化内容)
> 验证状态:⚠️ 首次(单次成立,跨任务待验)

---

## 一句话

要批量产**大量结构化内容**(N × 双语案件 / 条目),让多个子代理按**互斥分类**各写一批 **JSON**,再用**脚本统一校验 + 组装 + 编译检查 + 真机抽测**——人定 schema 与质检关,代理只填格子,组装交确定性脚本(零转义 bug)。

---

## 工作流(可复跑)

1. **定 schema + 互斥分类**:字段结构写死;给每个子代理一组**不重叠**的类别(避免跨批重复)+ 目标条数。
2. **子代理只产 JSON**:输出严格 JSON(最易机器校验),各写各的文件;别让它直接产最终代码格式。
3. **脚本校验**:Python 跑 schema 完整性 / 配比(如 scam:safe) / 查重(标题) / 计数。
4. **确定性组装**:`json.dumps(s, ensure_ascii=False)` 把每个字符串转成**合法 JS 字面量**——换行 / 引号 / 中文全自动转义,零手工拼接事故;再注入目标源文件。
5. **双关验收**:`node --check` 过语法 + 浏览器真机抽测渲染。

---

## 为什么这么做

- **并行** → 184 条的创作工时摊薄到多代理并发。
- **JSON 中转** → 校验 / 组装可机器化;`json.dumps` 让"数据 → 代码"零转义事故(手工拼字符串极易在引号 / 换行 / 中文上翻车)。
- **互斥分类** → 天然去重、保多样性。

---

## 边界

- ⚠️ 单次验证;质量上限仍取决于 schema 与质检关写得好不好。
- 适合"结构同构、量大"的内容;强叙事连贯 / 相互依赖的内容不适合拆并行。
