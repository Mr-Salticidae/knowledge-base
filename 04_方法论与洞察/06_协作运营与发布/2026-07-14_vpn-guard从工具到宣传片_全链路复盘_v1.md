# vpn-guard 全链路复盘：从一个防护需求，到开源工具，到宣传片

> 首次记录：2026-07-14
> 来源：一次会话内从"帮我做 VPN 防泄露防护"起步，一路做成开源工具 vpn-guard、上 GitHub、做封面、做 B站宣传片（占位粗剪）。全程人机共创。
> 状态：**规律已确立**；工具已开源（github.com/Mr-Salticidae/vpn-guard），宣传片工程私有存档（vpn-guard-promo）。

---

## 一句话总结

一个"需求→工具→开源→传播"的完整闭环里，最值钱的不是任何单点产出，而是几条反复复用的纪律：**交付前先证伪、开源前先脱敏自己、产线要可复现、蹭热点要克制、人机共创可以是暗线**。

## 弧线速览

1. **拉仓库**：出差回来同步 E 盘 29 个 git 仓库；分叉冲突用 merge 不 rebase；非自有仓库（无写权限）登记 `E:\.gitignore` 只拉不推。
2. **做工具**：VPN 出口一致性/防泄露审查（IP/DNS/WebRTC/IPv6 泄露 + 时区/语言一致性），做成 PowerShell 工具箱 vpn-guard。
3. **开源**：脱敏 → 可移植化 → 加 LICENSE/贡献指南/issue 模板/topics → 推 GitHub。
4. **封面**：MJ 底图 + 代码叠字合成社交预览图，照片底图转 JPEG 压进 1MB。
5. **宣传片**：复用 [[蒙眼剪辑法_方法论笔记]] 的"AI工具乐队"产线，edge-tts 旁白 + ffmpeg，出带旁白占位粗剪。

## 提炼的规律

### 1. 交付前实测证伪（本次头号教训 → 单独立律）

想用"TZ 环境变量让 Chrome 改时区"这个方案时，先写了个最小探针实测——**Windows 版 Chrome 根本不认 TZ 环境变量**，当场证伪，避免了交付一个"看起来对、其实静默失效"的假方案，改走 `tzutil` 切系统时区。同一纪律在本次触发三次（TZ 证伪、SAPI 被沙箱挡后转 edge-tts、git 推送后核实）。抽成 ⭐ [[交付前实测证伪律_v1]]。

### 2. 隐私工具，开源前先把自己脱敏（反讽自检）

vpn-guard 是"防泄露"工具，但初版 README 里写满了作者**自己的真实出口 IP 段、真实 ISP DNS、"人在中国"**等隐私——一个隐私工具开源时泄露了作者的隐私。规律：**任何工具开源前，先把它的价值主张施加于自身**。防泄露工具→先 grep 清掉自己的 IP/DNS/位置；防抄袭→先查自己有没有抄；教人省钱→先看自己方案贵不贵。推送前用 `grep` 扫一遍真实 IP/凭据/绝对路径，是硬步骤。

### 3. 可复现产线三支柱（从蒙眼剪辑法迁移并再验证）

宣传片工程沿用并再次验证了三条：**①配置逻辑分离**（时间轴/文案全在 `config.py`，改卡点不动逻辑）；**②音频先行**（先出配音→量真实时长写 `timing.json`→画面时长由它驱动，绝不手数秒）；**③占位→替换**（色卡/静图先占位验证节奏，真素材到位改一行路径重渲，时间轴不动）。本次新增一条本地化替代：ElevenLabs 无凭据时，**edge-tts（微软 Edge 神经 TTS，免费免密钥）**可作占位甚至成片配音，产线两种音频文件都吃。详见 [[蒙眼剪辑法_方法论笔记]]。

### 4. Windows + PowerShell 的编码坑（补进既有文档）

`.ps1` 含中文必须存 **UTF-8 BOM**，否则 PS5.1 按 GBK 读、中文全角标点解析失败；Python 往 GBK 控制台 `print` 一个 `✓`(U+2713) 直接崩（UnicodeEncodeError）。均属"所见非真相"同根，已补进 [[Windows下编码与DPI的所见非真相]]。另有：`[TimeZoneInfo]::Local` 在同进程内缓存，`tzutil` 切时区后要**新起进程**才读到（探针验证时踩过）；沙箱禁止对含 `C:\Program` 等系统路径的 `Remove-Item`（用带时间戳的全新临时目录、不删）。

### 5. 蹭热点的克制 framing：引用现象，不点名对象

想蹭"某 AI 公司封禁中国用户"的热度，但在国内平台点名=高危。定"A 档"：**只说"技术开始按地区给人划线了"这个现象，不点名**。既借了势（观众自己会联想），又保平台安全、保格调、保可否认性。规律：**借势敏感话题时，把矛头留在现象层，让观众在心里完成指认**——比直接开骂更高级也更安全。这是 [[小红书导流话术合规红线_扣1领资料_v1]] 之外的另一类"红线上跳舞"。

### 6. 人机共创可以是暗线，且要用真实器物证据

宣传片讲"技术该共享"，而它本身就是人机共创的。埋法：片尾不点破，只浮出 vpn-guard 真实 commit 里那行 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`——**这不是台词，是收据**。规律：meta/自反叙事要用**真实器物证据（receipt）而非摆拍**，经得起查，"懂的人会心一笑"的分量才立得住。
