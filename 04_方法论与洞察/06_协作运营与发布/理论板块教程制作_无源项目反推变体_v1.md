## 二、方法论沉淀

### [无项目可反推时，事实锚点降为「最小外部事实点 + 类比自洽」]

**核心**：理论稿没有 git 历史可锚，但不等于可以随便写。做法是**把需要外部核查的断言压到最少，其余全部走类比**——类比不承担事实责任，只承担「解释得通不通」的责任。

**来源**：本篇全文只有一处外部事实——vibe coding 的词源（2025 年初 Andrej Karpathy 提出）。其余「编程=给电脑写说明书」「三道门槛」「Chat 只说 / Agent 直接做」全是类比与分类性讲解，无数据口径风险。大纲的制作侧附注里专门列了「事实核查点」一项，把这唯一一处标出来供审核人核。

**操作规则**：
1. 起草时给每个断言分类：需外部核查的 / 类比性的。前者能压则压，压不掉的逐条列进大纲附注的「事实核查点」；
2. 涉及人名、年份、词源、数据口径的，宁可写模糊范围（「2025 年初」）也不编精确日期；
3. 类比的验收标准不是「像不像」，而是**「读者能不能用这个类比推出下一步」**——「机器人做饭」类比之所以留下来，是因为它能直接推出「代码要精确到每一步」。

**反例/边界**：如果理论稿里需要放榜单、市占率、模型跑分这类数据，本条不适用——那类内容必须走实测或标注数据源与统计口径。

### [纯理论板块的配图 = 统一示意卡片，源码入 git 留档]

**核心**：理论节没有界面可截。与其硬凑截图，不如**全量做示意卡片，并把卡片的 HTML + 共用 CSS 一起入库**——文案是要改的，改完能重出图，比留 PNG 值钱。

**来源**：本篇 6 张卡片共用一份 `base.css`，Chrome headless 批量截图；源码存 `reference/AI编程理论科普_卡片源码/`。风格直接对齐前三篇实战板块的班底卡，视觉上并入母文档不突兀。

**操作规则**：
1. 先定一份共用 CSS（底色、金色点缀、圆角卡片、字号梯度），各卡片只写内容层；
2. 分辨率对齐同母文档既有配图（本次 2400×1350）；
3. **逐张验收**：本次返工三处——HTML 连续空格被折叠（`v i b e   c o d i n g` 渲染成 `vibecoding`、`STEP 1` 变 `STEP1`，须用 `&nbsp;`）、标签宽度不齐、底部留白过多；
4. Windows 上用 Chrome headless 截图时，`--headless=new` + 绝对路径 file:// URL + 每次独立 `--user-data-dir` 才稳定；连续复用同一 profile 会静默不出图。

**反例/边界**：有真实界面可截的板块仍按「实拍优先」；示意卡片是补位手段，不是偷懒默认值。

### [理论节引言：范文骨架的「引出成品」功能句要换掉，不能硬凑]

**核心**：[[范文参考骨架律_v1]] 提炼的引言骨架是「勾兴趣 → 点 AI 趋势 → 引出成品」。理论节没有成品可演示，第三段功能句必须**换成等效功能**而不是硬找个东西来演示。

**来源**：本篇引言的实现是——「性价比最高的几分钟」勾兴趣 → 「门槛已从会写代码降到会说人话」点趋势 → **抛出三个问题**（编程在干嘛 / AI 怎么降的门槛 / 零基础凭什么能做出来）替代引出成品。既守住了系列节奏，又和另外三篇的开场不重样。

**操作规则**：功能骨架的每一句先问「这句在承担什么功能」，本板块没有对应素材时找**同功能的替代物**（问题清单、悬念、对照），而不是删掉这一功能或硬塞。

### [零基础扫盲：概念配生活类比，术语后置成小抄]

**核心**：面向完全没接触过编程的学员，抽象概念**一个都不能裸讲**。两条处理：① 每个概念配一个生活类比（说明书 / 装修队 / 发动机）；② 高频术语不在正文里逐个解释打断节奏，**统一后置成结尾的「词汇小抄」并声明"不用背、听到回查"**。

**来源**：本篇 14 个术语全部收进结尾小抄卡；正文里只在术语首次出现时给一句人话，不展开。这是 [[完结项目反推学员教程_大纲先行两级审核_v1]] 「减负三原则」中「准备工作后置成结尾卡片」在理论稿上的同构应用——**把打断节奏的东西一律后置**。

**操作规则**：
1. 正文里出现的每个术语，问一句「零基础读者读到这里会不会卡」，会卡就配类比或进小抄；
2. 小抄必须声明「不用背」，否则等于给学员加了记忆负担，与减负初衷相反；
3. 理论节结尾要**向后勾连**（本篇点明「后面所有实战都是『说→做→看→改』循环的重复」），让科普节成为后续实战的骨架线，而不是孤立的一节。

## 三、一句话结论

> 没有源项目可反推时，「大纲先行 + 审核后写正文」的主干不变，变的只是事实锚点——把需外部核查的断言压到最少、其余交给能推出下一步的类比；配图全量示意卡片且源码入库，因为文案一定会改。
