# H5 移植微信小程序:要点与合规红线

> 入档:2026-06-24
> 来源:把《反诈柜台》H5 移植成微信小程序(代码完成,发布因合规 / 流程成本搁置)
> 验证状态:⚠️ 首次;合规规则属平台事实

---

## 一句话

H5(DOM)移植小程序,**渲染层必须重写为 WXML/WXSS + `setData`(无 DOM),但纯数据 / 逻辑可原样复用**;动手前先撞**合规红线**——微信"游戏"须做**小游戏(需企业主体)**,小程序一般**禁游戏类目**。

---

## 移植要点

- **数据 / 逻辑零改**:从 H5 的 `script.js` 抽"数据段 + 逻辑函数",加 `module.exports` 即成小程序可 `require` 的模块;200 条案件库直接复用。
- **渲染层重写**:DOM 操作 → `setData` 驱动 WXML 模板;`<rich-text nodes>` 渲染带 `<b>` 的富文本;`<text>` + `white-space:pre-wrap` 保留 `\n`。
- **没有 Web 字体**:小程序加载不了 Google Fonts,标题回退系统衬线。
- **真机 / 逻辑可先验**:开发者工具模拟器能跑;UI 自动化跑不了(须桌面工具)。

---

## 合规红线(先查再动手,省移植工时)

- **"游戏"必须是小游戏**,小游戏须**企业 / 个体工商户主体**(个人号一般注册不了)。
- **小程序禁游戏类目**;科普 / 测验类**可走"教育 / 工具"类目**但仍有被驳风险。
- 发布链路:开发者工具建项目 + AppID → 上传 → 后台提交审核 → 发布,**全程需用户本人**(见 [[浏览器插件自动化的能力边界_v1]])。

---

## 如何使用

接"把 H5 搬上小程序":先判主体资质 + 类目准入,**再决定要不要投移植工时**;代码层按"数据复用 + 渲染重写"拆。
