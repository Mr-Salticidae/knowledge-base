## 待补丁条目（按发现日期）

### 补丁 1 · 封面是入场票的"第零层"

**触发**：异界气象台 minitest 封面 v1 翻车（2026-05-14）

**核心修正**：
- v1 的入场票框架假设了"前 0.5 秒识别"发生在视频内
- 实战中，**封面才是真正的入场票第零层**——观众在快手刷流里到不了点击，视频内的入场票就废了
- v1.1 必须明确区分"封面层入场票"和"视频内入场票"

**对应 SOP 修正**：
- 任何作品的封面必须当作"独立设计任务"处理
- 禁止从视频内截帧 + 套元素生成封面
- 封面 brief 必须在制作启动前由运营经理写明，**不放任制作侧自行决定**

### 补丁 2 · 封面 prompt 的五重反字幕约束（框架级原则）

**来源**：Cowork Claude 从「大地之上」与「异界气象台 v1」翻车踩坑提炼

**约束内容**：
> 封面层 MJ prompt 必须包含 `no text, no labels, no logos, no subtitles, no banners` 五重反约束。

**理由**：
- MJ 在 prompt 没强压制字幕的情况下，会按"它理解的央视纪录片画面"自带生成红条字幕
- 央视入场票应该通过**命名格式（"XX气象台"）+ 后期排版（思源宋体 Heavy 大字）双层叠加承接**，而不是让 MJ 在抽卡阶段自己生成文字元素
- 不只适用于央视格式预载——任何"自带制式画面"的素材（新闻联播、动物世界、气象预报）都有这个陷阱

### 补丁 3 · sref 强度的"视频-封面"差分

**来源**：Cowork Claude 异界气象台封面 v2 prompt 工程

**核心原则**：
> 封面层 sref 强度（sw）默认比视频层低 30-40。
> 视频用 sw 80 时，封面用 sw 40-50。

**理由**：
- 视频内画面追求**广角播报感**，需要 sw 80 来保证视觉连贯
- 封面追求**主体冲出框**，需要 sw 40-50 让 MJ 把主体推到画面前面
- 两者用同一个 sw 默认值会导致封面"画面满但主体小"

**适用范围**：所有沿用同一 sref 体系的作品。首次发现于异界气象台 minitest，但适用于所有 IP 系列化作品。

### 补丁 4 · 双轨诊断的判断窗口 ✅ 已升级为独立文档

**升级时间**：2026-05-15
**独立文档**：[[慢热作品识别_v1]]
**升级原因**：喂的慢热反转在 56h 窗口完整生命周期观察完毕（0.08% → 4.45% → 涨粉 19）——证据足够独立成文

**核心结论**（已沉淀到独立文档）：
- 24h 窗口的"流量虚假"判断对长内容 + 个人表达型作品存在错杀风险
- 收藏率 ≥ 0.1% + 时长 >90s + 个人表达型 = 慢热信号
- 任何"改作品"处方不能在 24h 内启动
- 投流追加在慢热筛查期（24-72h）内一律暂停

### 补丁 5 · 算法自动加推的盲点（待数据验证）

**触发**：凝视在快手"燎原计划 + 5 元投流"组合下，从 31 播放飙到 2979（×96），但点赞率从 6.45% 崩到 0.23%

**核心发现**：
- 快手算法对**完播率**极端敏感（凝视 15s + 9.7% 完播率 → 自动加推）
- 但算法**对"人群匹配度"几乎没识别能力**——它推了，但推错了
- 默认推荐模式 + 全勾选 = 把所有作品扔进算法盲池

**待补充条目**：
- v1 的"流量饥饿"判断处方是"换平台"
- 但实战中还有第三条路径："**接受被错误推送 + 用点赞率作为人群质量信号 + 不投流追加**"
- 需要写一条新的判断逻辑：被动加推 vs 主动投流的差异化诊断

### 补丁 6 · MJ 主体尺寸召唤词偏好（已验证 32 张实测）

**触发**：异界气象台封面 v2 抽卡（32 张产出实测）

**核心规律**：
> **单一主体（colossal / towering / monolithic）的尺寸召唤强度 > 群体（massive / countless / horde）**

**实证数据**：
- B-v1（鲸群 + massive whales）主体压倒感 2-3 分
- B-v2（单只巨鲸 + colossal whale）主体压倒感 5 分
- A 系列（单一沉钟塔 + towering tower）主体压倒感 4-5 分

**操作指南**：
- 封面 prompt 优先使用"单一主体 + 形容词激进"组合
- 如必须做群体，每个个体单独 prompt 控制，避免依赖群体词
- 此条同步入档到 `知识库/02_参数行为档案/MJ_v8.1_无oref_混合工作流.md`

### 补丁 7 · 所有目标比例必须 MJ 原生抽卡（流程层）

**触发**：异界气象台封面 v2 横屏比例返工

**核心规则**：
> **封面 brief 阶段必须同步规划所有目标比例。9:16 → 16:9 的事后处理（裁剪 / PIL 扩展 / 翻转）几乎都做不到原生构图质量。**

**事后处理 vs 原生抽卡的实测质量差**：
- PIL 画布扩展 16:9 左留白质量 = **1 分**（色块 + 噪点）
- MJ 原生重抽 16:9 左留白质量 = **5 分**（真实雾天 + 海面层次）

**操作指南**：
- 每个目标比例（9:16 / 3:4 / 16:9 / 4:3）必须有独立的 MJ prompt，**第一轮就抽出来**
- 运营经理写封面 brief 时必须明确写"四比例 MJ 原生 prompt 各 N 张候选"
- 不再默认制作侧会自己规划——把这条作为显式约束

### 补丁 8 · 副会话决策点的技术限制必须诚实披露（沟通层）

**触发**：异界气象台封面 v2 的 PIL 扩展翻车事件

**核心规则**：
> Cowork / 副会话在 AskUserQuestion 等决策点向跳蛛先生提供选项时，**必须诚实标注每个选项的技术局限**——不能用"技术上能跑通"代替"作品级质量"做自我安慰式验收。

**反例**：
- ❌ 选项写"水平翻转 + PIL 渲染"，未说明"扩展区是暗色色块不是真实图像"
- ✅ 选项写"水平翻转 + PIL 渲染（**扩展区是采样色块 + 噪点，非真实图像，质量上限有限**）"

**操作指南**：
- 决策点的每个选项必须附"技术局限标注"
- 自我验收时用"作品级质量"标准而非"技术能跑通"标准
- 受限时主动反馈，不要钻牛角尖凑合

### 补丁 10 · 作品-平台-受众三角匹配，不可能通吃四平台（顶层框架）

**触发**：异界气象台首日跨平台数据对比（5/15 凌晨）—— 快手 3070 / 抖音 76 / B 站 75 / 小红书 0 曝光，**四平台几乎完全不同的反应**

**核心修正**：
> 入场票框架 v1 假设了"入场票类型"是核心变量。但跨平台对比显示——**作品-平台-受众的三角匹配才是底层结构**，入场票类型只是其中一个变量。

**三角的具体表现**：

| 维度 | 控制变量 |
|---|---|
| 作品（入场票 + 概念密度 + 时长 + 视觉）| 创作者 |
| 平台（算法偏好 + 用户基数 + 文化语境）| 选择 |
| 受众（垂类浓度 + 接受门槛 + 互动习惯）| 涌现 |

**实证**：
- 凝视：快手 31 播放 vs B 站 1250 播放 + 208 赞（30 倍杠杆差）
- 同样美学辨识度入场票，B 站浓度高、快手浓度低
- 异界气象台：快手 3070（含燎原补贴）vs 抖音 76（算法实质判负）

**四平台不同的"内容契合区"**（待 v1.1 升级时正式表述）：
- **抖音** = 西瓜模式（文化预载 meme + 体裁反差 + 1-3 分钟）
- **B 站** = 凝视/R-07 模式（高视觉浓度 + 长内容 + 作品集）
- **快手** = 喂模式（个人表达 + 慢热 + 真粉转化）
- **小红书** = 教程模式（AIGC 知识 + 图文 + 思辨）

**操作铁律**（v1.1 升级时正式纳入 SOP）：
- 没有"全平台同步发"——任何单一作品试图通吃四平台 = 必然 3/4 平台浪费精力
- 每条作品立项时必须先确认"它的主战场是哪个平台"，其他平台只做归档
- "主战场"由作品-平台-受众三角匹配度决定，不由"我希望在哪个平台爆"决定

### 补丁 9 · minitest 不是科研任务，不为最优解（角色边界）

**触发**：跳蛛先生 2026-05-14 18:30 提醒"不是科研任务，不为最优解"

**核心修正**：
> 运营经理 finalize 决策时，不能把所有论证都建立在"实验干净性"上。**跳蛛先生是创作者 + 独立游戏开发者，不是研究员**。他的目标是好作品 + 活账号 + 可持续节奏，不是论文。

**反例（修正前）**：
- 把 9 个 finalize 决策全部论证为"为了保护实验干净性"
- 把 H1/H2/H3 假说真伪当作 minitest 的主要指标
- 用"科学家心态"压制"够用就好"的实用判断

**操作指南**：
- minitest 的真正第一指标是**涨粉转化率**（账号资产累积），不是假说真伪
- 数据观察的优先级：涨粉 > 完播率 > 点赞率 > 播放量 > 假说指标
- 当论证某个决策时，先问"如果完全不考虑实验干净性，这个决策还成立吗"——如果成立，就用作品判断/运营常识的语言重新论证；如果不成立，就要重新审视是否被"科学家心态"绑架了
- 运营经理的本职是**把好作品送到该看的人面前 + 让账号资产持续累积**，不是把作品当实验素材

**为什么这条最关键**：
> 它修正的不是某个方法论细节，而是**运营经理这个角色本身**。当运营经理开始用"实验干净性"论证一切时，已经偏离了角色本职。所有上层补丁（1-8）在没有这条补丁约束的情况下，可能滑向"为了完美实验而过度设计"的陷阱。
