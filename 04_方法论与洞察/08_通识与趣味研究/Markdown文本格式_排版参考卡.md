# Markdown 文本格式：排版参考卡

> 入档:2026-07-21
> 来源:CommonMark 规范 / GitHub Flavored Markdown(GFM)
> 性质:标记语言 · 写作工具 · 文本格式化
> 定位:「通识与趣味研究」方向——写作者随时可查的速查卡片,覆盖日常 90% 排版需求

## 一句话

**Markdown 用极简符号把结构「写」进纯文本:标题、列表、加粗、链接、代码……语义与格式合一,源码即排版,一处编写到处渲染。**

## 核心格式速查

### 1. 标题(Heading)

```markdown
# H1 一级标题
## H2 二级标题
### H3 三级标题
#### H4 四级标题
```

> 要点:建议在 `#` 后留一个空格;一篇文档通常只用一个 H1。

### 2. 文字强调(Emphasis)

| 写法 | 效果 | 用途 |
|---|---|---|
| `*斜体*` 或 `_斜体_` | *斜体* | 轻微强调、外来语 |
| `**加粗**` 或 `__加粗__` | **加粗** | 核心关键词、重点句 |
| `~~删除~~` | ~~删除~~ | 标记废弃/修改痕迹 |
| `***粗斜体***` | ***粗斜体*** | 强强调(较少用) |

### 3. 列表(List)

**无序列表**(用 `-` / `*` / `+` 均可):

```markdown
- 第一项
- 第二项
  - 子项(缩进 2 空格)
  - 子项
```

**有序列表**:

```markdown
1. 第一步
2. 第二步
3. 第三步
```

> 要点:嵌套列表缩进 2 或 4 空格;列表前后留空行才能正确渲染。

### 4. 链接与图片

```markdown
[链接文字](https://example.com)
[带标题的链接](https://example.com "鼠标悬停文字")
![图片说明](https://example.com/image.png)
```

> 要点:链接 `[]` 在前、`()` 在前;图片在链接前加 `!`;站内引用常用相对路径。

### 5. 引用(Blockquote)

```markdown
> 一级引用
>> 嵌套引用
```

效果:

> 一级引用
> > 嵌套引用

### 6. 代码(Code)

| 类型 | 写法 | 效果 |
|---|---|---|
| 行内代码 | `` `code` `` | `code` |
| 代码块 | ` ```语言 ` 包裹 | 语法高亮块 |

````markdown
```python
def hello():
    print("Hello, Markdown!")
```
````

> 要点:代码块标注语言名(python / markdown / bash 等)可触发语法高亮。

### 7. 表格(Table)

```markdown
| 表头 A | 表头 B | 表头 C |
|---|---|---|
| 左对齐 | 居中 | 右对齐 |
|:---|:---:|---:|
| 内容 | 内容 | 内容 |
```

| 表头 A | 表头 B | 表头 C |
|---|---|---|
| 左对齐 | 居中 | 右对齐 |
|:---|:---:|---:|
| 内容 | 内容 | 内容 |

> 要点:对齐通过 `:` 位置控制——左 `|:---|`、中 `|:---:|`、右 `|---:|`。

### 8. 分割线(Horizontal Rule)

```markdown
---
```

效果:

---

> 要点:三个或以上 `-`、`*` 或 `_`,前后留空行。

### 9. 复选框(Task List,GFM)

```markdown
- [x] 已完成
- [ ] 待办
```

效果:

- [x] 已完成
- [ ] 待办

### 10. 转义(Escape)

当符号本身是文字时,用 `\` 转义:

```markdown
\* 这不是斜星号 \*
\# 这不是标题 \#
```

效果: \* 不是斜体 \* 、\# 不是标题 \#

## 排版好习惯

- **空行即分段**:段落之间必须空一行,否则会被合并到同一段。
- **符号后加空格**:`# 标题` 优于 `#标题`,多数解析器要求 `#` 后跟空格。
- **一致性**:同级列表统一用 `-`,不混用 `-` 与 `*`。
- **少即是多**:能用结构(标题、列表)讲清的,就不用加粗、斜体堆砌。
- **源码可读**:Markdown 的卖点是「源码即排版」——写的时候就该看着整齐,不必依赖渲染。

## 如何渲染

- **本地编辑器**:VS Code(内置预览)、Typora(所见即所得)、Obsidian
- **在线渲染**:GitHub / GitLab / Notion / 飞书文档 / 语雀
- **静态站点**:Hugo / Hexo / Astro / VitePress 直接消费 `.md`
- **编程转换**:Python `markdown` 库、Node `marked`、Rust `comrak` 等
