# SKILL 入门完全指南

> 从"Skill 是什么"到"怎么用起来"，一篇读完。

---

## Skill 是什么

把 Skill 想象成一本**岗位手册**。

你雇了一位助理（AI），它很聪明，但每次上班前都失忆了——不记得你喜欢怎样的格式，不记得那套你反复讲的规则，也不记得上次它踩过的坑。Skill 就是你给它的那本手册：角色是什么、遇到什么情况怎么处理、最后交付什么格式、哪些话绝对不能说。

手册在，每次上班都能立刻进入状态。手册不在，就得从头教。

**具体形态**：Skill 是一个 `.md` 文件，文件开头有一段简短的自我介绍（YAML 格式），告诉 AI 这份手册叫什么名字、在什么情况下应该拿出来用。后面的正文就是具体的工作规则。

```yaml
---
name: aigc-prompt-optimizer
description: 把口语化的创作需求，翻译成适合 Midjourney、Seedance 等工具的专业提示词...
---
```

---

## 什么时候值得用 Skill

当你发现自己**反复在教 AI 同一件事**，就值得做成 Skill。

最典型的三类场景：

**Prompt 优化类**：你总要把自己的模糊想法改成 Midjourney 能读懂的专业描述，每次都要重新说一遍规则。

**工作流辅助类**：某个任务有固定步骤，比如"笔记→视频分镜→Remotion 脚本"，这套流程每次都一样，适合固化。

**工具语法类**：特定工具有特定写法，比如 Midjourney 的参数格式、Seedance 的镜头描述惯例，不想每次都查文档。

**不建议一开始就碰的类型**：需要安装 Python / Node / FFmpeg 才能跑起来的复杂 Skill，或者说明文档很少却声称"一键完成所有事"的。新手阶段，越简单越好。

## 怎么让 Skill 工作起来

### 方法一：SKILL_INDEX 挂载（推荐）

这是目前最稳的方式，**不需要安装任何东西**。

背景解释：Cowork 每次启动时会生成一个随机的路径编号（UUID，像一串乱码），导致"安装"进去的 Skill 下次找不到。就像你把手册锁进了一个每天换地址的保险柜——没法用。

解决办法是：**把手册放在知识库里，让 Claude 知道去哪找**。知识库的路径是固定的，不会每次变。

知识库里有一份目录文件叫 `SKILL_INDEX.md`，里面登记了所有 Skill 的名字、触发词和文件位置。就像图书馆的索引卡：你告诉管理员（Claude）"索引卡在哪"，它就能自己去查、自己去取书、自己开始工作。

**使用流程，只需两步：**

**第一步**：每次开始新对话时，发这句话给 Claude：

```
你好，我的知识库在 D:\AIGC工作站\知识库，
请先读取 SKILL_INDEX.md（路径：D:\AIGC工作站\知识库\SKILL_INDEX.md），
后续对话中自动判断何时调用哪个 skill。
```

**第二步**：正常说你的需求，Claude 会自动识别应该调用哪个 Skill。

触发可以很精确，比如直接叫名字："用 aigc-prompt-optimizer 优化这份 prompt"；也可以很自然，比如说"帮我复盘一下这次测试"——Claude 会匹配到 aigc-postmortem 这本手册。

> **实测结果**：2026-06-04，在一次虚拟形象设计对话中，aigc-prompt-optimizer 和 aigc-postmortem 两个 Skill 均被正确识别并完整执行，无一遗漏。这套机制已验证可用。⭐⭐

**唯一的注意事项**：Claude 每次开新对话都会"失忆"，所以每次新对话都要发一次开场白。把这句话存到手机备忘录或输入法快捷短语，粘贴一下就行，几秒钟的事。

### 方法二：手动安装（备用）

如果你用的是 Codex 或 Claude 客户端，也可以手动把 Skill 放到对应目录。

以 Codex 为例，目录是：

```
%USERPROFILE%\.codex\skills
```

在这个目录下新建一个文件夹，文件夹名字用 Skill 的 `name`，然后把 `SKILL.md` 放进去：

```
.codex\skills\prompt-master\SKILL.md
```

Claude 客户端同理，只是根目录换成 `.claude\skills`。

安装后需要**完全退出并重启客户端**，Skill 才会在新对话里生效。

## 安装前先检查三件事

### 这个文件是 Skill 吗？

真正的 Skill 文件通常满足：文件是 `.md`，开头有 `---`，YAML 里有 `name:` 和 `description:`，正文讲清楚触发条件和处理流程。如果只是一篇普通教程或 README，不一定能直接当 Skill 用。

### 它有没有附属文件？

有些 Skill 会引用额外的参考文件，比如：

```
references/templates.md
references/patterns.md
```

如果只拿了 `SKILL.md`，这部分就缺失了。SKILL_INDEX 挂载方式的好处之一是：知识库里可以把 Skill 和它的附属文件一起存好，不会漏。

### 它需要什么环境？

纯文本类的 Skill（写 prompt、生成结构、复盘模板）没有额外依赖。视频、剪辑、代码类的 Skill 可能需要 FFmpeg、Python、Node 等工具——安装 Skill 只代表 AI 知道了流程，不代表你的电脑已经准备好了跑这个流程。

## 第一次怎么测试

不要一上来就丢大项目。第一次测试的目标只有三个：AI 知道这个 Skill 什么时候该用、输出格式符合规定、它没有把普通回答伪装成 Skill 执行。

推荐这样触发：

```
请使用 prompt-master skill，把下面这个口语化需求优化成适合 Midjourney 的提示词：
我想做一张高端雪山单板广告图，要有电影感、速度感、很贵的感觉。
```

好的测试不是看它说得多漂亮，而是看它有没有遵守 Skill 里的核心规则。

## 日常使用的正确姿势

**明确点名**：第一次用某个 Skill 时，直接说名字。不要只说"帮我优化一下"，这样 Claude 不一定能判断要不要调 Skill、调哪个。

**给足输入**：Skill 不是读心术。至少要告诉它：目标工具是什么、用途是什么、有什么约束、你手头有什么素材。越专业的 Skill，越需要清晰的输入。

**一次只测一个目标**：不要第一次就说"Midjourney、Seedance、Suno、剪辑脚本、发布文案全给我"。拆成步骤，哪步出问题就修哪步，也方便复盘。

## 常见坑位

**坑一：把 Skill 当能力升级包**。Skill 是工作手册，不是给模型装新技能。它能提升稳定性和复用性，但不能让模型突然拥有它本来没有的能力。

**坑二：复杂 Skill 只装了主文件**。引用了附属文件的 Skill，单独拿 `SKILL.md` 是跑不完整的。

**坑三：忘了存档版本**。Skill 升级后行为可能变化。跑通的版本建议存一份到 [[07_skill存档索引]]，至少记下版本号和测试日期，效果变差时可以回退。

## 判断一个 Skill 好不好

用这个小清单对着看：

- 它是否说清楚了什么情况下触发？
- 它是否知道什么时候该问问题、什么时候直接输出？
- 它是否给出了稳定的输出格式？
- 它是否写了禁区（哪些事不做、哪些话不说）？
- 它是否适配了具体工具，而不是泛泛地说"更专业"？
- 2-3 次测试，输出质量是否稳定？

真正好的 Skill，不是看起来复杂，而是能让你少解释、少返工、少重开会话。

## 新手推荐路线

```
第 1 步：选一个单文件 prompt 优化类 Skill（依赖少，容易看懂）
第 2 步：把开场白存好，每次新对话发一下，挂载 SKILL_INDEX
第 3 步：用一个小任务测试，确认格式和触发都对
第 4 步：记录测试前后对比
第 5 步：把跑通版本存档到 07_skill存档
第 6 步：再尝试视频 / 剪辑 / Remotion 等复杂 Skill
```

最值得沉淀的不是别人写好的 Skill，而是你自己的创作流程——你怎么描述画面、怎么拆分镜头、怎么检查角色一致性、怎么把粗糙想法变成可执行提示词。这些才是真正属于你的能力包。
