# 概念锚定 · 风格置换迭代法 v1

> 入档：2026-06-12
> 来源：[[2026-06-12_对话甲骨文二进制获奖图复盘]]（复盘记录仓库）
> 验证状态：⚠️ 首次发现（1 次获奖验证），暂时性结论

---

## 一句话总结

**当出图"概念对但不美"时，冻结叙事核心层、整体置换风格基底层——不要在原风格里修修补补。**

---

## 案例

「对话」获奖图三轮迭代，叙事核心层（两个时代 + 同构动作 + 字符交汇）**一字未动**，动的只有风格层：

| 版本 | 风格基底 | 结果 |
|---|---|---|
| v1 | cinematic photography + chiaroscuro + volumetric light | 概念成立，审美不达标 |
| v2 | editorial illustration 词系整体置换 | "好很多"（仅剩构图问题）|
| v3 | 同 v2 基底 + 软过渡修正 | 获奖 |

v1 的失败不是概念问题——是摄影词系把图拖向了"史诗概念图"的拥挤审美。诊断错了层，会在错误的层里无限修补。

---

## 操作规则

### 1. 先诊断：不满意的是哪一层？

- **概念不对** → 改叙事核心（主体/动作/关系）
- **概念对但不美** → 换风格基底，叙事核心冻结

### 2. 风格词系成组进出，不可混写

| 词系 | 成员 | 互斥关系 |
|---|---|---|
| 摄影词系 | cinematic / volumetric light / chiaroscuro / shallow depth of field / photorealistic / color grading | 与插画词系互斥 |
| 扁平插画词系 | flat color blocks / thin precise linework / editorial illustration / controlled detail density / isometric | 残留一个摄影词都会打架 |

**换基底 = 旧词系全部清除 + 新词系整组进入**。v1→v2 时 volumetric/chiaroscuro/dof/photorealistic 一次性全删。

### 3. 新基底优先移植已验证参考图的原 prompt 词汇

自己发明风格形容词命中率低；**参考图的原 prompt 词组是已经被验证能产出该风格的词**，直接移植（本案移植了 `clean isometric editorial illustration / polished editorial style / controlled detail density`）。

`--sref [参考图URL] --sw 200` 是并行备用路线；本案纯文字移植已达标，未动用。

### 4. 负面词同步加防回滚锚

换插画基底 → `--no photorealistic, 3d render`（防止 glowing 类残留词把图拖回写实渲染）。

**负面词不是固定模板，是迭代记录——每一版的负面词都是上一版病灶的疫苗**（本案 v3 又追加 `hard split line, frame border` 根除 v2 的硬切）。

---

## 适用边界

- 适用：概念已被确认成立（用户/自己明确"概念好，画面不好看"）
- 不适用：概念本身没立住——换十种风格也救不了错的概念
- 不适用：只差局部细节——那是 Vary Region 的活，不要整体重抽
