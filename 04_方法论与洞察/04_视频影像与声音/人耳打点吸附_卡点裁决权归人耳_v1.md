# 人耳打点吸附 —— 卡点裁决权归人耳，毫秒精度归算法

> 来源：50 号案例《学员作品集混剪·The Banjo King》卡点迭代（2026-07-02）
> 验证状态：✅ 单项目强验证（v1 纯算法被否 → v2 人耳打点吸附获用户"非常完美"）

## 一句话律

**卡点剪辑中，"选哪些拍"必须归人耳，"精确到哪一毫秒"归算法：让创作者跟着音乐打拍子，再把每个手打点吸附到 ±0.18s 内最强能量攻击峰、提前 1 帧补视觉延迟。**

## 问题链（三层失败逐层暴露）

1. **v0 纯 onset 检测**：切点普遍晚于人耳感受的攻击 100–380ms（能量流量峰滞后于 attack 起点）→ 用户反馈"没敲在鼓点上"。
2. **v1 算法吸附攻击峰 + 提前 1 帧 + 冲击力度特效**：时机对了，但仍被否——[[蒙眼剪辑法_方法论笔记]]早有记录：**算法分不清预备拍/主拍、乐句边界/鼓点**。装饰音密集的曲子（本例班卓琴），吸附会钉在装饰音上。
3. **v2 人耳打点 + 算法吸附**：命中。

## 操作流程

1. 给创作者一个**浏览器打点器**（单 HTML：AudioContext 播放 + 空格打点 + 导出秒列表；见 50 号项目 `scripts/卡点标记器.html`）。引导语：先完整听一遍不打；第二遍只在"画面必须换"的重击上打，宁缺勿滥。
2. 手打点典型误差 +30~160ms（轻微滞后）——**不修正手速，只做吸附**：每个点取 ±0.18s 窗口内未平滑能量冲击（RMS 帧差半波整流）最大处。
3. 吸附结果再**提前 1 帧**（30fps 下 33ms）：人眼处理比人耳慢，画面须略早于声音才"感觉同步"。
4. 手打点自然携带**乐句级结构**（密刀段/长镜段/冲刺连击），这是算法给不出的额外收益——直接决定各段镜头时长分布。

## 边界与反例

- 创作者若节奏感弱（打点自身漂移 >±0.25s），吸附窗口会抓错峰；此时退回"算法初标 + 人逐点审"。
- 窗口 ±0.18s 适合中速重卡点（~108 BPM）；快曲（>150 BPM 半拍 <200ms）窗口须收窄，否则会吸到邻拍。
- 打点体验本身有娱乐性（用户评"像节奏医生/节奏天国"）——工具化时可往节奏游戏方向包装。✅ 已落地：见 [[BeatTapper卡点标记器说明]]（2026-07-02 升级为正式 web 应用，上线蛛网之上「可玩」板块）。
