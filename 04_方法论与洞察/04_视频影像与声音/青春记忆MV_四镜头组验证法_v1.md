# 青春记忆 MV · 四镜头组验证法 · v1

> 入档:2026-05-26
> 触发:一次青春记忆型 MV 从 Suno 主版本、角色/教室基准、故事板、视频片段转码归档一路跑到首批视频片段
> 性质:叙事型青春 MV 的阶段复盘 —— **先验证记忆证据链,再扩大全片生产**

---

## 一句话总结

**青春记忆型 MV 不要先追完整剧情,要先锁 4 个"记忆证据镜头":核心意象、亲近动作、普通课堂动作、现在时闭环。** 这 4 个镜头成立,全片的情绪骨架才成立;这 4 个镜头不成立,继续批量生成只是在扩散错误。

---

## 本阶段探索成果

### 1. 主题从"白月光"落到"细节证据"

这类青春记忆 MV 的有效方向不是拍告白、分手、重逢、落泪,而是拍:

- 水点甩到镜头前;
- 递纸巾时手指擦过;
- 作业纸边擦过手腕;
- 试卷传递时指尖停在一道题旁;
- 窗边讲题时纸页、手、玻璃反光同时存在。

关键洞察:**白月光不是被解释出来的,是被细节证明出来的。**

这让 MV 避开了青春题材最容易俗套的"大情节",转向更耐看的"小动作 + 记忆残留"。

### 2. B2 音频只改变节奏,不推翻故事板

Suno 方案从 B 切到 B2 后,最重要的判断是:**不需要整体重画故事板。**

B2 的价值是时长更松、停顿更多、剪辑压迫感更低。它改变的是镜头呼吸,不是叙事结构。

经验教训:当音乐版本升级时,先判断它改变的是哪一层:

| 改变层 | 处理方式 |
|---|---|
| 情绪主旨变了 | 重写故事线 |
| 段落结构变了 | 重排分镜 |
| 只有时长/呼吸变了 | 保留故事板,调整镜头长度 |

这次属于第三类。

### 3. 四镜头组优先级

本阶段没有从 12 个故事板镜头平均推进,而是选出 4 个核心镜头组:

| 镜头类型 | 作用 | 验证什么 |
|---|---|---|
| 核心意象镜头 | 主记忆镜头 | 片名意象、人物距离、纸面与窗光 |
| 手部关系镜头 | 亲近动作镜头 | 亲近但没说破的第一视角关系 |
| 普通课堂动作镜头 | 记忆证据镜头 | 普通动作如何变成记忆证据 |
| 现在时闭环镜头 | 情绪收束镜头 | 未完成、放回原位、情绪释放 |

这 4 个镜头对应一条最小叙事闭环:

```
她在窗边讲题 → 我记住手和光 → 普通试卷动作被放大 → 多年后把题放回窗边
```

**先验证这条闭环,比先铺满所有分镜更有效。**

### 4. 第一视角 POV 是风格,也是约束

本项目的 POV 规则不是装饰,而是叙事机制:

- 男主不能作为完整人物出现;
- 男主不能露脸;
- 只能出现手、手腕、纸边、桌角、笔记本边缘、镜头停顿;
- 喜欢不是台词,是镜头反应。

这条约束把"青春暗恋"从偶像剧情节拉回到真实记忆:观众看到的不是男主在表演喜欢,而是镜头在记住她。

### 5. 720 教室舞台图的正确用法

教室 720 全景图很有价值,但不能无脑作为角色镜头参考图。

它适合做:

- 人工理解空间结构;
- 统一桌椅、窗户、黑板、窗帘、暖光;
- 写 prompt 时提取空间词。

它不适合直接塞进每个角色镜头作为强参考,因为会把全景图的鱼眼、超广角、360 空间畸变带进成片。

经验教训:**全景舞台图是"空间母版",不是"镜头参考图"。** 对角色近景,更稳的是用文字重建教室空间,只把女主脸部基准图 + 身份板 + 对应故事板图作为图像参考。

### 6. liblib 原始导出必须先工程化转码

liblib.tv 默认导出的 MP4 对剪辑并不总是友好。项目里沉淀了 `convert_liblib_video.py`,把原始导出统一转成:

- H.264 / AVC
- 1080p 以内,保持比例
- 24fps
- yuv420p
- AAC 音频(如原片有音频)
- `+faststart` MP4

本阶段已验证该转码规格能输出剪辑友好的 1920x1080 / 24fps / H.264 / yuv420p 视频片段。

归档规则也同步明确:兼容版进入项目的当前视频片段目录,原始下载文件进入日期归档目录。这样主线只保留可剪辑资产,原始文件仍可追溯。

## 可复用流程

### 阶段 1 · 先写"不拍什么"

青春题材很容易被大情绪吞掉。先列禁区:

- 不告白;
- 不分手;
- 不重逢;
- 不落泪;
- 不用大段字幕解释;
- 不把白月光拍成梦幻符号。

禁区越清楚,小动作越有空间。

### 阶段 2 · 建立三层时间

```
现在时:安静、少动作、颜色淡
回忆时:动作具体、色温暖、细节密
记忆放大:水点/纸巾/纸页/指尖/窗光交叉剪
```

三层时间的作用是防止 MV 变成纯线性剧情。青春记忆的质感来自"现在看见物件 → 回忆被唤起 → 细节被放大 → 回到现在放下"。

### 阶段 3 · 选 4 个验证镜头

不要平均推进全片。先选:

1. 一个片名意象镜头;
2. 一个关系动作镜头;
3. 一个普通动作证据镜头;
4. 一个现在时闭环镜头。

这 4 个通过后,再扩展成完整分镜组。

### 阶段 4 · 每个视频片段先转码再入库

```
liblib 原始导出
  → convert_liblib_video.py
  → H.264 / 1080p / 24fps / yuv420p
  → 进入当前视频片段目录
  → 原始导出进入日期归档目录
  → 更新视频片段清单
```

工程规范不是收尾工作,而是剪辑阶段能不能顺畅展开的前置条件。

## 反向陷阱

### 陷阱 1 · 用"青春感"替代具体动作

`青春感、白月光、遗憾、校园、阳光` 这些词都太大。真正能落地的是"手指停在第三步旁"、"纸边擦过手腕"、"纸巾被她抽走"。

**抽象词必须翻译成可拍动作。**

### 陷阱 2 · 把空间母版当强参考图

720 全景舞台一旦直接作为强参考,会把"空间理解"变成"镜头畸变"。全景图适合辅助写 prompt,不适合支配近景镜头。

### 陷阱 3 · 一上来追全片完整度

全片完整但核心 4 镜不亮,等于没有记忆锚。先把 4 镜打透,再进入批量生产。

### 陷阱 4 · 原始视频直接进剪辑线

HEVC / 高帧率 / 非 yuv420p 的素材在剪辑软件里可能出现卡顿、颜色/兼容性问题。生成视频入库前必须转成统一规格,否则后面每一步都在还债。
