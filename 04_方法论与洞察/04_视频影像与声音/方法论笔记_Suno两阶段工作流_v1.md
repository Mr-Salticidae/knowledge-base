# Suno 两阶段工作流 · Simple→Custom · v1.1

> 入档：2026-05-17 / v1.1 升级：2026-05-19
> 触发：23《再少年》音乐共创副会话 5 轮迭代后的关键发现 → 29《扔掉》二次验证（3 轮跑通 + Simple Mode 95% 命中）
> 性质：Suno 创作工作流的**最优路径默认值**（已二次验证，可正式确立为 22_Suno 实验室核心方法论）
> 关联：[[Suno_v5.5_行为规律]] / [[识别工具天花板的时机]] / [[22_Suno实验室]] / [[Vocal_Gender反选_风格prior_v1]] / [[Suno版权过滤器规避_v1]]

## 一句话

**Simple Mode 求 happy accident（发现），Custom Mode 锁定固化（执行）**。这是 Suno 创作工作流的最优路径，不应该"直接上 Custom"。

---

## 触发场景

23《再少年》副会话原本按 brief 的"直接 Custom Mode"工作流跑。**第一版（v0 Simple）跑出来后跳蛛先生不满意**，副会话切换路线：

```
[v0] Simple Mode 跑 → 不满意
 ↓
[v1] Simple Mode（路线调整 → 改写为纯主题描述求最大惊喜）
     → 跑出男女对唱 happy accident ⭐
 ↓
[v2] Custom Mode（首次切换 → 锁定男女对唱结构 + 一唱一答）
 ↓
[v3] Custom Mode（押韵补丁 + 字符压缩）
 ↓
[v4] Custom Mode（注入口琴贯穿动机 + 段落差异化）→ 收口 ✓
```

**核心发现**：v1 的男女对唱是 Suno 自己跑出来的**意外**——但这个意外比所有规划好的方案都更对。**没有 Simple Mode 那一轮，永远不会发现这条最优路径**。

---

## 两阶段工作流核心机制

### 阶段 1 · Simple Mode 求 Happy Accident

**目的**：让 Suno 在更松的约束下**自由组合**，发现你没想到的可能性。

**操作**：
- 只给"主题描述"（不进 Custom Mode）
- 让 Suno 自由选 style + structure + lyrics
- 跑 3-5 个变体
- **核心判断**：有没有任何一个变体出现了"我没想到但很对"的元素？
  - 不同声部组合（如男女对唱）
  - 意外的乐器选择
  - 出人意料的节奏
  - 意外的歌词处理

**适合的人**：
- 接受"探索性"的人
- 愿意为"意外发现"付 3-5 个变体的成本

**不适合的情况**：
- 已经 100% 知道想要什么
- 项目极度赶时间（探索性投入 = 30 分钟以上）

### 阶段 2 · Custom Mode 锁定固化

**目的**：把 Simple Mode 发现的好东西**精确锁定**，避免后续迭代时丢失。

**操作**：
- 进入 Custom Mode
- Style of Music 明确锁定"那个发现"
- Lyrics 用英文 `[Male] [Female] [Both]` 标签强制结构
- More Options 设 Weirdness 30-35% + Style Influence 75%
- 用 advanced 模式（Edit Lyrics / Replace Section / Inpaint）做精细调整

**关键**：Custom Mode 是**执行工具**，不是发现工具。**所有"应该是什么样"的判断都应该在 Simple 阶段做完**。

---

## 为什么"直接 Custom"是次优解

### 反直觉发现

**约束越多，发现空间越小**。Custom Mode 把所有参数都填好后：
- Suno 严格按 prompt 执行
- 减少了"它自己创造"的可能性
- 你只能看到"你想象中的东西"，看不到"你没想象的东西"

### 案例验证

23《再少年》如果直接 Custom：
- v0 指令文件按"男主女辅"写 → Custom Mode 会严格按这个跑
- **永远不会跑出"男女对唱一唱一答"** ← 实际的最优解
- 跳蛛先生永远不会发现这条路径

**Simple Mode 的"松"反而是优势** —— 它给了发现空间。

## 例外情况 · 什么时候可以跳过 Simple

✅ 可以直接 Custom 的场景：

1. **已经有非常明确的 reference song**（如要做"周杰伦《手写的从前》风格的歌"）→ 此分支的具体拆解法见 [[参考曲复刻法_reference曲到Suno提示词_v1]]
2. **二次迭代同一首歌**（已经有 v1 满意版，只是要做 v2 小调整）
3. **极度赶时间**（< 30 分钟必须出 demo）

❌ 必须用两阶段的场景：

1. **新项目第一次创作**
2. **跨流派探索**（如这次"古风流行 + 男女对唱"是新组合）
3. **追求"出人意料的好"** —— Simple Mode 是必经之路

## 与既有方法论的关系

### 上位母方法论

[[识别工具天花板的时机]] 的应用案例之一：

- Custom Mode 的天花板 = 严格执行 prompt，**无法发现 prompt 外的可能**
- Simple Mode 接管 = 发现层（找出 prompt 该写什么）
- Custom Mode 退一步 = 锁定层（把发现的写进 prompt）

**两阶段工作流 = 给 Custom Mode 的"识别天花板 + 重新分工"**。

### 与 Suno auto-write 的关系

[[22_Suno实验室/02_作品/003_不算就不算]] 验证的 "Lyrics 留空 + Write with Suno" 工作流是**两阶段工作流的子模块**：
- Write with Suno = 让 Suno 在 lyrics 这一层"发现"
- Custom Mode 的 Lyrics 字段 = 用人写的 lyrics 锁定

两个机制同构：**让 Suno 发现 + 用人来锁定**。

## 适用边界

### 适用场景

- ✅ Suno 音乐创作（所有 Suno 项目）
- ✅ 可能推广到其他生成模型（如 MJ 的"参数松 → 发现 → 参数严 → 锁定"）

### 不适用场景

- ❌ 完全 deterministic 的工具（如 PIL 图像处理）
- ❌ 流水线项目（确定流程不需要探索）

## v1.1 升级 · 29《扔掉》二次验证（2026-05-19）

### Simple Mode 命中度的进一步提升

23《再少年》Simple Mode 命中度 60%（跑出男女对唱意外）。
29《扔掉》Simple Mode 命中度 **95%**（直接出完整歌词骨架，比手写更好）。

**两个案例的命中度差异分析**：
- 23 的 prompt 是"主题描述"（IP 内核 + 情境 + 调性），让 Suno 在多维度自由组合
- 29 的 prompt 包含一条**结构性语法描述**（详见下方"结构性语法 prompt"补丁），约束维度更窄但更精确

### 结构性语法 prompt（v1.1 关键新增）

**新发现**：在 v5.5 上，**结构性语法描述比直接给歌词文本更有效**，因为它给了 Suno"长肉"的语法骨架而不是约束它。

29《扔掉》Simple Mode prompt 里这一行触发了 95% 命中：

```
lyrics juxtapose past-tense unfinished actions with present-tense
already-arrived states ("X hasn't fallen yet / I've already arrived at Y")
```

Suno 没有得到任何具体歌词文本，但自己长出了七次变奏：
- 黑外套还没落地 / 我已经跑过十年
- 风还没吹到脸上 / 我先把自己交给浪
- 城市还没完全远 / 我已听见潮声来
- ……

**机制**：给 Suno "歌词的语法结构 + 一个英文范例"，它能跨语言反推出大量符合该语法的中文意象。这比直接给中文歌词更"留出长肉的空间"。

**适用条件**：
- 你能用一句话描述歌词的语法骨架（如时态对位、问答结构、列表式排比、否定累积等）
- 你能给一个英文范例帮 Suno 理解
- 你愿意让 Suno 自己长意象

**对照旧做法（直接给歌词文本）**：
- 老路径：写完整歌词草案 → Suno 严格执行 → 你只能看到你能想到的
- 新路径：写结构性语法 → Suno 长出你没想到的意象 → 跳蛛先生选最好的

### 操作流程更新（替换原"第二步"）

```
原 v1：
  第二步：Simple Mode 跑 3-5 版（不要进 Custom Mode）

v1.1 升级：
  第二步：Simple Mode 跑 3-5 版（不要进 Custom Mode）
    └─ 子选项 a：用"主题描述" prompt（适合多维探索）
    └─ 子选项 b：用"结构性语法" prompt（适合追求 95% 高命中）⭐ 新
```

### Vocal Gender 反选作为后续 Custom Mode 阶段的调味

29 项目同时浮现了 [[Vocal_Gender反选_风格prior_v1]] 这条新方法论——这是 Custom Mode 阶段的调味技巧，**与两阶段工作流正交独立**，可以叠加使用。

### 两案例汇总对比

| 维度 | 23《再少年》 | 29《扔掉》 |
|---|---|---|
| Simple Mode 命中度 | 60% | **95%** |
| 迭代轮次 | 5 大轮 | **3 大轮 + 1 hook 替换** |

| Custom Mode 难点 | 字符压缩 + 段落差异化 + 贯穿动机 | 仅一次 hook 替换（版权过滤器） |

| 工作流验证状态 | v1 沉淀 | **v1.1 二次验证确认** |
