# cinemagraph 微动作：待机层是真实感引擎

> 验证等级：⚠️ 首次发现（单次受控实验，数据扎实但需跨题材复验，不得作铁律）
> 场景：图生视频 / 动态壁纸 / cinemagraph 微表情 ｜ 实测工具：可灵 Kling 3.0 Turbo
> 源实验：2026-06-23 含泪凝视眨眼壁纸 A/B/C 三组对照（见复盘记录同名长文）

## 核心规律

让 AI 静态肖像"活起来"，**真实感主要来自"待机层"，而非主动作本身**。

- **待机层**（贯穿全程、prompt 必写）：湿润眼球反光、泪光/高光微颤、眼神几乎不可察觉的微漂移与重新聚焦、呼吸带来的极轻起伏、碎发被气流轻拂。这一层承担绝大部分"活着"的错觉。
- **主动作**（如眨眼）：是配角，宁少勿多。实测一次软眨只造成 +3 亮度、~0.5s 的微小起伏，感知贡献很低。

## 三条可操作结论

1. **待机层 > 主动作**：写微动 prompt 先把待机层写足，主动作只点一下即可。零主动作（纯活着凝视）和有主动作版本同样真。
2. **循环壁纸选"零主动作"版**：壁纸是循环播放，"N 秒一次"的主动作循环后会变成机械节拍（=最常见的假感来源）。无主动作版无断点、可无限循环、接缝最好做。**单条欣赏≠循环壁纸，用途决定取舍。**
3. **主动作要"快"不要"慢"**：真人眨眼闭合仅 0.1–0.15s。写"缓慢/温柔眨眼"会得到困倦机械的假眨；要写"快速、闭合短促、略不对称、半无意识"。

## 配套要点

- **可灵 3.0 系列界面无负向提示词框**（官方网页/App + liblib 第三方均为单一正向框；英文资料说的"支持负向"指第三方 API 的 negative_prompt 参数）。约束一律用**正向句**写："不要 X" → "正在发生 Y"（如"全程保持睁开"代替"不要连续眨眼"）。
- **可灵能听懂绝对秒数**（⭐）："第 3 秒眨一次"实测落在 3.08–3.21s，误差 ~0.1s。精确编排微动作可用秒数卡点。
- **工作流**：Turbo 1080p 调参挑 seed → 视频 3.0 标准版出 4K（seed 不跨模型，4K 为重新生成）。
