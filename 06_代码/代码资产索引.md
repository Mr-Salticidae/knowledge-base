> 入档:2026-05-09(原文件名 README.md → 2026-05-13 改名为 代码资产索引.md,避免与根目录 README 在 Obsidian 文件树里同名)

## 为什么有这个目录

跟字体一样,代码也是**跨项目复用的资产**。每个项目有自己的"就近开发"位置(`{项目}/代码/...`),但一旦稳定下来,就在这里维护一份**统一镜像**作为索引。

好处:
- 找历史代码不用记"放在哪个项目子目录" — 来这里全局检索
- 发现哪些设计模式在不同项目间重复(如水印、封面合集、视频拼接)
- 切换电脑或交接项目时只需要同步这一个目录

**镜像规则**:
1. **只镜像知识库外的项目代码**(`{AIGC工作站}/02_*\` / `06_*\` / `11_*\` / `14_*\` / `临时创作\` 等)。
2. **知识库内的代码不镜像**(例如 `05_视觉系统/檐下系列封面模板.py`)——它已经在 vault 里,Obsidian 双链直接 `[[檐下系列封面模板]]` 即可,无需再复制一份;否则会出现"知识库镜像知识库"的路径冗余(2026-05-13 清理了 `06_代码/知识库/` 这个历史嵌套)。
3. **本目录只复制不开发**。所有改动应在原项目目录下做,然后同步过来。当前同步是手动的(脚本如下),未来可考虑加 git hook 自动化。

```bash
# 同步脚本(可放到 _sync.sh)
# 跑前确认所有项目里 .py 都已落盘
rsync -av --include='*.py' --include='*/' --exclude='*' \
  --exclude='__pycache__' --exclude='.git' --exclude='.obsidian' \
  --exclude='Lora_GAZE_数据集/' \
  D:/AIGC工作站/{02_,06_,11_,14_}*/  D:/AIGC工作站/代码/
```

## 跨项目复用模式备忘

通过这次集中,可以看出跳蛛先生作品集里的几个核心代码模式:

### 1. 双字体协奏:中文衬线 + 英文 ExtraLight 无衬线

最早出现在 11_凝视 的 footer,在 14_我的一天 沉淀为标准:
- 中文:`SourceHanSerifSC-ExtraLight`(克制宋体)
- 英文:`DejaVuSans-ExtraLight`(极细无衬线)
- 加 `letter-spacing` 让字符散开 — 摆脱方块感的关键

### 2. supersample 抗锯齿

11_凝视 / 14_我的一天 都用 2x 渲染再 LANCZOS 缩回 — 在 PIL 默认抗锯齿基础上额外提升细字号清晰度。

### 3. 智能裁剪保人脸

`make_xhs_cover.py` 的从上往下裁取 30% 偏移,保留人脸在画面 1/3-1/2 区域。

### 4. 项目结构惯例

`{项目}/原图/` + `{项目}/成品图/` + `{项目}/代码/{用途}/` + `{项目说明}.md`(项目级文档,如 [[MV模板说明]])。14_我的一天 严格按这个惯例归档。

### 5. 长任务的分块兜底

「这瓜包熟吗」(2026-05-12) 第一次撞到沙箱单条命令 45 秒上限,ffmpeg 跑不完 135 秒音频的 MV。沉淀的解法在 `MV模板/make_mv_chunked.py`:
- 把长任务按时间切片,每片单跑一次 ffmpeg
- 关键技巧:zoompan 表达式用全局帧号 `(start_frame + on) / total_frames` 校正,镜头跨片连续
- 最后用 `concat demuxer + -c copy` 无重编码拼接,几乎不耗时
- 适用于任何「单次命令超时但任务可时序切分」的场景,不只 MV

### 6. 横版 → 竖版用「项目封面切片」而非模糊填充

「这瓜包熟吗」(2026-05-12) 同日做抖音版 9:16,先试了主流的"主视频居中 + 同画面模糊 + 暗化"路线,三轮调参都摆脱不掉"两个图层叠在一起"的视觉违和。结论:同画面的清晰+模糊本质上是冲突的,参数调不掉。

改用的方案沉淀在 `MV模板/make_vertical_cover.py`:
- 不动主视频内容,只是裁中央方块
- 上下边框直接从**项目已有的 3:4 海报封面**切出"标题条 + 页脚条"
- 深底色从封面采样,自动同色系
- 静态边框预烤成一张 PNG,每帧只做一次 overlay → 比"每帧 4 个 overlay + 高斯模糊"快 7 倍

适用前提:项目有配套的 3:4 海报封面(里面有标题排版 + 品牌徽章)。如果还没出过海报,先用 `05_视觉系统/` 里的模板出一张。

### 7. 静态合成元素要预烤成 PNG

`make_vertical_cover.py` 的提速核心:渲染时不要每帧重算静态内容。把"深底色 + 标题条 + 页脚条"先合成一张 `frame_template.png`,然后视频帧只做"裁主视频 + 一次 overlay 到底板"。

这个套路适用于任何"动态视频 + 大块静态装饰"的场景——比如带固定字幕条的解说视频、带 logo 角标的直播录屏、带固定边框的图卡序列。能不能用的判断:**装饰部分在整个时长里都不动**就行。

### 8. AI 视频原始导出先转码,再进主线

生成平台导出的 MP4 不等于剪辑友好素材。`MV模板/convert_liblib_video.py` 的规则:

- 入库规格统一到 H.264 / 1080p / 24fps / yuv420p / faststart;
- 兼容版进入当前项目的视频片段目录;
- 原始下载文件进入日期归档目录;
- 每次入库后更新项目的视频片段清单。

这条规则的价值是把"生成结果"和"剪辑资产"分开:前者可追溯,后者可直接进入时间线。

## 添加新代码的规则

1. **就近开发**:新代码在项目目录里写,不在本目录直接写
2. **稳定后镜像**:项目交付后(像 14_我的一天 这次),把 .py 同步到本目录对应位置
3. **更新本 README**:在"代码 → 作品 映射"表
