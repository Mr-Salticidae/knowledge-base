# BeatTapper 卡点标记器说明 —— 人选拍机器定毫秒的 web 打点工具

> 来源:50 号案例的单 HTML 雏形(`scripts/卡点标记器.html`) → 2026-07-02 升级为可分发 web 应用
> 开发主线:`{AIGC工作站}/tools/beat-tapper/`(**改动在那边做,这里只镜像**)
> 线上:蛛网之上「可玩 · Plays」板块 `/beat-tapper/`(public 直拷模式,单文件稳定型)
> 方法论母档:[[人耳打点吸附_卡点裁决权归人耳_v1]]

## 是什么

浏览器里的音乐打点器:加载本地音频 → 跟着重击按空格打点 → 每个点吸附到 ±0.18s 内最强攻击峰 → 导出秒级时间戳(复制 / JSON / CSV),交给蒙眼剪辑法的剪辑脚本。纯前端单文件零依赖,音频全程本地处理不上传,手机触摸可用。

包装成节奏游戏体验(用户对雏形的评价"像节奏医生/节奏天国",本档即把该娱乐性做成正式产品):打点即时判定 PERFECT(≤45ms)/GOOD(≤90ms)/还行(≤180ms)/自由拍 + 连击计数;波形上黄线=吸附后切点、灰点=原始手打;回放检验模式重放音乐,在每个打点处闪光+咔哒声,用耳朵验打点准不准。

## 实现要点(可跨项目复用)

1. **能量流量法 onset 检测搬进浏览器**:与 `snap_user_taps.py` 同源——单声道混音 → 5ms 窗 RMS 包络 → 正向差分半波整流为攻击强度 → 手打点在 ±0.18s 窗内取最大值。Web Audio `decodeAudioData` 拿到 PCM 后纯 JS 循环即可,103s 曲目毫秒级算完,**无需任何后端/依赖库**。
2. **打点计时用音频时钟**:`AudioContext.currentTime - t0 + offset`,不用 `Date.now()`——不受页面渲染卡顿影响,判定阈值(45/90/180ms)才有意义。
3. **回放咔哒声用 Web Audio 精确调度**:`osc.start(when)` 一次性把全部打点的 click 排进音频时间线,不是 setTimeout 逐个触发;即使 UI 线程卡顿,听感检验依然毫秒准。
4. **判定只是手感反馈,不影响精度**:游戏化(判定/连击/闪光)负责"好玩",吸附负责"准"——两层解耦,关掉吸附也能导出原始手打值。
5. **合成已知答案的测试音频做验证**:JS 里现场生成"底噪 + 已知时刻噪声 burst"的 WAV 喂给加载流程,断言吸附误差为 0ms——工具类 web 应用可以不靠人耳做回归。

## 踩坑记录

- **`BufferSource.onended` 竞态**:暂停→立刻重播时,旧 source 的 onended 异步晚到,会把播放状态误重置为 0。解法:回调里比对 `mySrc !== src` 只认"当前 source"的自然播完。
- **预览环境 rAF 冻结**:后台 iframe `visibilityState=hidden` 时 `requestAnimationFrame` 完全不跑,时间显示/回放闪光在自动化验证里全是死的,`preview_screenshot` 也超时——正是 [[Claude预览环境不派发滚动事件_滚动类功能无法行为验证_v1]] 的又一次同族再验;退级用 preview_eval 驱动状态机 + DOM 快照完成验证。
- 编辑器/管道会把 `﻿` 转义**规范化成不可见 BOM 字符**,字符串里要 BOM 用 `String.fromCharCode(0xFEFF)` 全 ASCII 写法最稳(CSV 导出给 Excel 认 UTF-8 用)。
