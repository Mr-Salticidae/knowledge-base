## 这是什么

每次创作探索后,把**有 insight 的发现**沉淀到这里,形成可复用的长期资产。

不是产出归档,**是 insight 归档**——
图、音频、视频等素材本身通常不在本仓库,这里只存"我学到了什么"。

本目录是一个 **Obsidian Vault**(配置见 `.obsidian/`)。所有文档之间用 `[[双链]]` 互相连接,既能点击跳转,也能在 Obsidian 的关系图谱里看到知识脉络。

## 怎么用这个知识库

1. **从这份 README 出发** —— 它本身就是入口地图(MOC, Map of Content),按主题串好所有文档
2. **按目录找** —— GitHub 浏览优先看 [[03_prompt模板库索引]] 和 [[04_方法论与洞察索引]],再进入子目录
3. **沿着 `[[双链]]` 跳** —— 每篇文档底部的"关联文档"段都是入口,顺着走能看到一整条思路
4. **按 `Ctrl+G` 看图谱** —— Obsidian 的关系图谱会直观显示哪些文档是中心、哪些是孤岛
5. **搜不到就建** —— 写新文档前先 `Ctrl+O` 模糊搜一遍,确认没有就建;能加补丁就别建新文件

### Frontmatter 写法

### 怎么选颜色(决策树)

2. **是可复用的 prompt 段或获奖图复盘?** → 🟢 `类型/prompt模板`

6. **是角色一致性主线的金字塔级方法论?** → 🔴 `类型/核心方法论`
7. **其他风格 / 媒介 / 审美方法论** → 🔵 `类型/风格审美`(兜底类)

### 颜色与"主题脉络"的关系

上文 MOC 里的"主题脉络一~九"是**阅读视角**(按使用场景串),颜色是**视觉视角**(按文档类型一眼分类)。两者会有交叉:比如 [[图生视频_ForwardOnly原则]] 在 MOC 里归"视频/影像方法论",颜色按"IP视觉"涂(因为这条线主要服务 IP 视频出片)。这是正常的 —— 一个文档可以在 MOC 里出现多次,但**颜色标签只有一个**。

### 改类型 / 加新颜色

- 改类型:直接改 frontmatter 的 `tags`,Obsidian 重新索引后图谱颜色自动跟着变
- 加新颜色:本规范暂定 7 色封顶。如果将来开了新主线(如"抖音视频化战术"、"商业化"、"AI 编程工程化"等 AIGC 工作站新延展),先想能否归到现有 7 类。**真的需要新增**时,同步更新本表 + `.obsidian/graph.json` 的 `colorGroups[]`

## 链接规范(写新文档必读)

### 1 · 每篇文档必须有"关联文档"段

放在文档末尾,**用 `[[]]` 双链语法**,不要写裸路径或 \`code\` 包裹的文件名(那样在 Obsidian 里点不开)。

**好的写法:**

```markdown
## 关联文档

- 顶层方法论:[[角色一致性金字塔]]
- 互补技巧:[[装扮签名vs五官精度]]、[[personalize与moodboard分工]]
- 主线作品:`{AIGC工作站}/03_StyleCreator暗调版\...`  ← 仓库外的产物可用裸路径
```

**不好的写法(老格式,Obsidian 点不开):**

```markdown
- 关联方法论:`04_方法论与洞察/01_角色一致性与视觉签名/角色一致性金字塔.md`
- 关联现象:`personalize与moodboard分工.md`
```

### 2 · `[[]]` 的几种写法

| 写法 | 效果 |
|---|---|
| `[[文件名]]` | 跳转到该文档 |
| `[[文件名\|显示文字]]` | 显示别的文字,点击同样跳转 |
| `[[文件名#二级标题]]` | 跳到该文档的某个标题 |
| `![[文件名]]` | **嵌入**该文档的全部内容到当前文档 |
| `[[文件名]]` 但目标不存在 | Obsidian 会标灰,点击可直接创建该文档 |

### 3 · 链接的方向(枢纽 + 卫星)

- **卫星 → 枢纽**:具体档案(sref 档案 / 复盘)在底部链回**顶层方法论**
- **枢纽 → 卫星**:顶层方法论列出**典型例证文档**
- **同层互链**:类似主题的方法论(如 [[AI甜妹脸vs复古东方美人]] 和 [[装扮签名vs五官精度]])互相链接

### 4 · 仓库外的引用保持裸路径

主线作品在 `{AIGC工作站}/` 别处时,继续用 \`{AIGC工作站}/xxx\\\` 这种裸路径写,不要硬塞 `[[]]`(Obsidian 链接只在 Vault 内有效)。

### 5 · 待写文档也可以先 `[[]]` 占位

```markdown
- 顶层工具:[[StyleCreator方法论]] ← 待写
```

Obsidian 会把它显示为灰色"未解析链接",点击就能直接创建,等于自动维护一份"待写清单"。

### 6 · 命名约定

- 文件名**不带 .md 后缀写在 `[[]]` 里**:正确 `[[角色一致性金字塔]]`,错误 `[[角色一致性金字塔.md]]`
- 文件名**不重名**:即使在不同子目录,文件名也保持全局唯一(Obsidian 双链按文件名解析,重名会冲突)
- 命名风格:**核心概念名 + 限定词**,例如 `seed_行为规律` / `personalize与moodboard分工` / `MJ审核避雷词表`
- **`README.md` 全库只能有一个**:就是当前这份根目录的 MOC。任何子目录想做"目录说明"的,文件名用 `{业务名}索引.md`(如 [[代码资产索引]])或 `{业务名}说明.md`(如 [[MV模板说明]])——这样 Obsidian 文件树里不会出现多个 README 同名节点,图谱里也不会出现"两个 README"的歧义

1. **每个文件独立,不要做大目录索引**——索引由本 README 统一承担
