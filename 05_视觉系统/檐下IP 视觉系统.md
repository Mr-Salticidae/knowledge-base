# 「檐下」IP 视觉系统

> 入档:2026-05-03
> 性质:作品集长期 IP 的稳定性护城河
> 首次实战:《花期很短，她很慢》(5/4 15:00 发布)

---

## 这是什么

「檐下」是跳蛛先生小红书账号的核心 IP 立柱。一个长期 IP 需要**稳定的视觉签名**(印章 + 字体 + 排版),否则每张封面的"风格抖动"会稀释 IP 心智。

GPT Image 2 / MJ 等生图模型每次都会"自由发挥"——**这是 IP 视觉一致性的天敌**。
代码生成则是 100% 可控的工艺:
- 像素级保真原图
- 字体 / 字号 / 列距 / 印章位置 完全一致
- 印章是同一枚红方框 + 同一个篆书"檐下"

→ 所以《檐下》系列每一张封面都"长得像一家人"。

---

## 核心组件

### 1. 阳文(朱文)篆书印章「檐下」
- 红边框 + 红色篆书字 + 中央留白(透明)
- 文人雅士款,典雅克制,不喧宾夺主
- 印泥不均、笔画断裂、微旋转 — 老印章质感
- 印面分布:每个字单独撑满"自己的格子",视觉权重均衡

### 2. 三列竖排楷书诗
- 字号:H / 34 — 小到像批注,不像标题
- 列距:char_size × 1.50 — 紧凑而不拥挤
- 起始 y:H × 0.53 — 与肩部齐,避开人脸
- 字色:温白 (250, 244, 232, 240)
- 软阴影 + 微弱发光,文字"嵌入"画面而非"贴在"画面上

### 3. 调色
- 暗部抬冷(蓝紫 +0.014)
- 高光抬暖(琥珀 +0.025)
- 轻晕影 — 视线收向中心
- 整体微提对比 ×1.05
- **极轻**——目的是"修饰"原图而非"覆盖"原图

## 使用方法

```bash
# 1. 改 檐下系列封面模板.py 顶部的配置:
#    - SRC: 输入原图路径
#    - DST: 输出路径
#    - COLUMNS: 三列诗句(右→左)
#    - SEAL_CHARS: 印章二字(此 IP 固定为"檐下")
#
# 2. 运行
python3 檐下系列封面模板.py
```

30 秒出封面,**所有「檐下」系列共享同一套视觉语言**。

## 适用场景

- ✅ 《檐下·初见》、《檐下·暗面》系列首图
- ✅ 任何「檐下」IP 主线作品(古风少女 + 留白 + 诗)
- ⚠️ 非「檐下」题材(如未来的《肌理》《拾光》)需要做新模板,**不要复用此印章**
- ❌ 教程类、实操类、信息密集型贴文(印章会显得过于文人感)
