## 原始 prompt 参考

```text
conceptual comparison of two AI assistants, one represented as a precise software engineer operating a terminal and test pipeline, the other represented as a visual productivity assistant arranging files, web previews and creative assets, clean isometric editorial illustration, clear visual contrast, modern workstation environment, polished educational article style, controlled detail density
```

---

## 图本身的赢点

### 1. 抽象差异被翻译成了两套工作台

这张图没有直接写"Codex vs 视觉助手"之类的标签,而是把能力差异转译成工作场景:

- 左侧:终端、代码屏幕、键盘、测试管线、工具零件
- 右侧:文件卡片、网页预览、创意素材、图像资产、桌面整理

这比在人物旁边写文字说明更高级。观众不用读标签,只看工作台就能理解:左边偏工程执行,右边偏视觉整理和生产力协作。

### 2. 左右分屏让比较关系一眼成立

画面中间有清楚的纵向分界,左右背景色也分开:

- 左侧偏粉紫,空间更靠近代码工作站
- 右侧偏蓝紫,空间更像视觉资产桌面

这个分屏结构让"比较"先成立,再让观众看细节。如果没有这条明确分界,两位助手可能会变成同一办公室里的两个角色,比较关系会弱很多。

### 3. 两个角色相似,但任务差异清楚

两侧角色都使用无表情面具、机械手、露出机械结构的头部,保持"同一 AI 助手族群"的统一感。差异没有放在人脸或服装上,而是放在手上的任务和周围资产上。

这让图像既统一又有对比:角色是同类,能力方向不同。这个处理比做成两个完全不同的 IP 更适合教育文章配图。

### 4. 教育文章感来自受控细节密度

prompt 里写了 `controlled detail density`,成图确实没有把屏幕、文件和桌面堆满。细节集中在可读的功能物件上:

- 左侧屏幕有代码/终端感,但没有生成真实文字
- 右侧墙面有网页和图像预览,但没有密密麻麻的 UI
- 桌面上有工具、文件、杯子等生活化锚点,但不抢主旨

这让图能作为文章头图或知识卡片使用,不会变成概念设定图或杂乱的信息图。

### 5. 柔和配色降低了工具比较的攻击性

如果用高对比红蓝、强烈科技黑、赛博霓虹,这张图会变成"工具对决"。现在的粉紫 / 蓝紫低饱和调色让它更像教育性比较,不是阵营站队。

这对 AI 工具类文章很重要:画面表达的是"分工",不是"谁碾压谁"。

## 可迁移方法论

### 能力边界要物件化

**核心**:解释两个工具或角色的差异时,不要只写性格标签,要把能力边界转成可见的工作物件。

**来源**:本图把"软件工程助手"翻译成终端、测试流水线、代码屏幕;把"视觉生产力助手"翻译成文件、网页预览和创意资产。

**验证状态**:首次发现 / 待验证

**操作规则**:
1. 先为每个对象写一句能力定义。
2. 再为每个能力定义列 3-5 个工作物件。
3. 让物件出现在手边、屏幕、墙面或桌面上,不要只当背景装饰。
4. 尽量避免直接生成文字标签,把说明交给物件关系。

**反例/边界**:
- 如果读者完全不了解对象,仍可能需要后期加标题或图注。
- 如果比较对象超过 3 个,单张图会拥挤,更适合做表格或多图组。

### 对照图先建秩序,再加细节

**核心**:概念对照图的第一优先级是比较结构清楚,细节只能服务这个结构。

**来源**:本图先用左右分屏和背景色建立对照关系,再用终端、网页预览、文件资产填充差异。

**验证状态**:首次发现 / 待验证

**操作规则**:
1. 先指定 `conceptual comparison`、`clear visual contrast`。
2. 用左右分屏、上下分区或两套工作台锁定比较结构。
3. 每一侧只保留能代表该对象的核心物件。
4. 用统一画风、角色造型或视角防止两侧割裂成两张图。

**反例/边界**:
- 如果目标是剧情插画,过强分屏会显得像信息图。
- 如果需要表达协作而非对照,可以改成同桌工作或同一流水线。

### 工具定位图适合低饱和教育插画

**核心**:AI 工具类文章配图不一定要炫技;低饱和、干净等距、受控细节更适合承载方法论表达。

**来源**:本图使用 clean isometric editorial illustration 和 polished educational article style,让概念可读性优先于视觉奇观。

**验证状态**:首次发现 / 待验证

**操作规则**:
1. 用 `clean isometric editorial illustration` 锁定教育图解感。
2. 用 `polished educational article style` 把画面从游戏设定图拉回文章配图。
3. 用 `controlled detail density` 限制杂乱。
4. 配色避免强赛博对抗,优先低饱和双色分区。

**反例/边界**:
- 如果用于封面点击率测试,可能需要更强的第一眼冲击。
- 如果用于产品官网主视觉,角色和界面可能需要更精确的品牌识别。

## 可复用 prompt 母版

> 适合做"两个工具 / 两类助手 / 两种工作流"的概念对照图。

```text
conceptual comparison of two {tools or roles}, left side represented as {role A} operating {3-5 concrete work objects}, right side represented as {role B} arranging {3-5 concrete work objects}, clear split-screen visual contrast, both characters share the same AI assistant design language, modern workstation environment, clean isometric editorial illustration, polished educational article style, low-saturation two-color background, controlled detail density, readable object relationships, no text labels, no logo, no watermark
```

### 本案例稳定版

```text
conceptual comparison of two AI assistants, left side represented as a precise software engineer operating a terminal, code editor, keyboard and test pipeline, right side represented as a visual productivity assistant arranging files, web previews, image boards and creative assets, clear split-screen visual contrast, both assistants share a calm humanoid robot design with mechanical hands and simple mask faces, modern workstation environment, clean isometric editorial illustration, polished educational article style, soft pink and blue color blocks, controlled detail density, readable object relationships, no text labels, no logo, no watermark
```

### 关键变量

- `conceptual comparison`:锁定"对照",不是普通双人办公图。
- `left side / right side`:强制分屏结构。
- `operating terminal and test pipeline`:让工程助手的任务可见。
- `arranging files, web previews and creative assets`:让视觉生产力助手的任务可见。
- `same AI assistant design language`:避免两侧角色割裂。
- `controlled detail density`:保证文章配图可读性。

---

## 下次同类图的行动清单

1. 先写比较对象的"能力定义",再写画面。
2. 每个对象只选 3-5 个最能代表能力的物件。
3. 对照关系优先用构图和色块表达,少依赖文字。
4. 如果用于文章头图,保留低饱和和受控细节;如果用于社媒封面,再增加视觉冲击。
5. 成图后检查:观众不看 prompt,是否能一眼看出两边工作不同。
