## 图本身的赢点

### 1. 把抽象工作流翻译成物理机制

这组图没有直接画"prompt 变成 skill"的文字说明,而是把流程变成一个压缩装置:

```text
重复输入 → 中央压缩 → 稳定手册
```

这比普通信息图更有效。观众能看到"为什么 Skill 是复用机制",而不是只看到一个漂亮的文件图标。

### 2. 主体关系比装饰更重要

第 0 张赢在关系完整。气泡不是散落装饰,而是正在进入压缩系统;手册不是孤立物件,而是系统输出物。

这说明 Remotion 外部资产不应该只追求更漂亮,而要补足程序图形最难表达的"概念关系"。

### 3. 纸面颗粒降低了程序感

MJ 输出里的纸面颗粒、手绘线条和轻微不完美,让图不再像普通 SVG 卡片。它保留了知识视频需要的解释性,同时多了一层编辑插画质感。

对 Remotion 来说,这类图适合作为关键 scene 的视觉锚,再由 Remotion 叠加字幕、标题和时间线。

### 4. 留白让它能进入视频管线

主图下方没有塞满信息,这对视频合成很关键。Remotion 可以在下方叠字幕,也可以保留安全区,不需要裁切画面主体。

这是一条重要筛选标准:适合单图欣赏的 MJ 图,不一定适合进入 Remotion;能进 Remotion 的图必须给字幕、标题和运动留位置。

## 可迁移方法论

### 抽象流程要先找物理隐喻

**核心**:解释工作流、系统或能力沉淀时,先找一个可见的物理机制,再出图。

**来源**:本案例把"重复 prompt 沉淀成 Skill"翻译成压缩漏斗。

**验证状态**:首次发现 / 待验证

**操作规则**:

1. 先写清楚流程的输入、转换、输出。
2. 为转换环节找一个物理机制:漏斗、蒸馏器、流水线、图书馆索引、压缩机。
3. 让输入和输出都保持可识别,不要只画抽象光效。
4. 如果用于视频,预留字幕安全区。

### Remotion 的外部资产要补概念关系,不是补装饰

**核心**:MJ / Seedance 应该补 Remotion 原生图形难以表达的视觉隐喻,而不是单纯把背景做漂亮。

**来源**:本案例第 0 张提升最大的地方是"气泡被处理成手册"这条因果线,不是材质或光效。

**验证状态**:首次发现 / 待验证

**操作规则**:

1. 每个外部资产先回答:它补的是主体、关系、动作,还是氛围。
2. 如果只补氛围,优先不接入,避免破坏 Remotion 可控性。
3. 主图必须能服务 scene goal,不能只是"更好看"。
4. 接入后仍由 Remotion 管字幕、标题、节奏和最终包装。

### 组图决策优先看接入价值

**核心**:同一组 MJ 图都好看时,主图选择不按单张审美排序,而按生产用途排序。

**来源**:第 2 张动势更强,但第 0 张因果链和 `SKILL` 锚点更完整,所以第 0 张适合作主图;第 2 张适合作 Seedance 动作参考。

**验证状态**:首次发现 / 待验证

**操作规则**:

1. 主图看语义完整度。
2. 动态参考看运动方向和能量。
3. 风格参考看线条、颗粒和调色。
4. 进入 Remotion 的图必须有字幕安全区。


### 下一轮优化方向

如果继续迭代,不要重写全部 prompt,只改三个变量:

1. 强化 `compression funnel outputs directly into the handbook`,减少装置和手册之间的断裂感。
2. 保留 `paper grain texture` 和 `thick crisp dark outlines`,不要转成光滑 UI 风。
3. 用 `single readable label: SKILL only` 限制乱码文字。

## 如何使用

1. Remotion 预览时先使用主图作为 `scene_04` 的 foreground image。
2. Seedance 2.0 生成视频时,优先用第 1 张或第 2 张做视觉/动作参考。
3. 后续如果做全片混合资产,每个 scene 都按"主图 / 动态参考 / 风格参考"三类归档。
4. 只把能增强概念关系的资产接入 Remotion,纯装饰图不进主线。
