# 网易云音乐发布 Brief · 通用模板

> 入档：2026-06-03
> 性质：可复用的发布前 intake 模板 + 一次性喂给 LLM 的输入清单
> 配套：[[网易云音乐人发布SOP_v1]]（决策框架"怎么填"）—— 本篇是"发起一次发布要收集什么 / 产出什么"的填空表

---

## 一句话

发布一首新歌时，**先用这份 Brief 把输入一次性收齐，再让 LLM 按模板自动补全标签 / 文案 / 封面方向 / MV 方向**——把"每次发布临时想字段"变成"填空 + 一键生成"。具体每个字段**怎么填、怎么避坑**走 [[网易云音乐人发布SOP_v1]]，本篇只管"要哪些字段 + 产出什么"。

---

## 如何使用

1. **发起新歌发布时**，先填「项目基础信息 + 歌曲文件」两段，把素材和定位收齐。
2. **把填好的 Brief + 歌词 + （可选）音频/封面素材一次性发给 LLM**，让它按下方"网易云上传内容 / 文案系统 / 专辑包装"逐项产出。
3. **产出的标签 / 文案，用 [[网易云音乐人发布SOP_v1]] 的决策框架做二次校验**（曲风标签别选「实验/其他」、情绪标签同光谱、专辑描述 ≠ 主创说等）。
4. **发布后**按"发布后动作 + 复盘"段收集评论与数据，为下一首 / MV 开发反哺。

> 模板与 SOP 的分工：**Brief = 收什么 / 产出什么（清单）**，**SOP = 每个字段怎么填、为什么这么填（决策）**。两份配合用，不要二选一。

---

## 一、项目基础信息

**必填**

- 歌曲名称：
- 演唱形式：独唱 / 男女对唱 / 合唱
- 风格参考：参考歌曲 + 参考歌手

**可选**

- 创作背景
- 希望传递的情绪
- 目标听众

## 二、歌曲文件

- 音频文件（MP3 / WAV）
- 歌词文件（TXT）
- 封面素材（可选）

---

## 三、网易云上传内容（需 LLM 产出）

| 项 | 规则 | 需要产出 |
|---|---|---|
| 1. 曲风标签 | 最多 2 个；优先推荐算法匹配，避免过于小众 | 标签推荐 + 推荐理由 |
| 2. 情绪标签 | 最多 3 个 | 标签推荐 + 推荐理由 |
| 3. 场景标签 | 最多 3 个 | 标签推荐 + 推荐理由 |

> 各标签可选项与避坑（如曲风别选「实验/其他」、情绪同光谱、场景看真实收听场景）见 [[网易云音乐人发布SOP_v1]]。

## 四、文案系统（需 LLM 产出）

| 项 | 用途 / 目标 | 需要产出 |
|---|---|---|
| 4. 主创说 | 发布后置顶展示；让听众理解歌曲并互动 | 长版 + 短版 |
| 5. 私信通知粉丝 | 新歌发布通知；提高首日播放量 | 简洁版 + 情感版 |
| 6. 站内笔记 | 歌曲动态页发布；获得评论与分享 | 标题 + 正文 + 话题标签 |

## 五、专辑包装（需 LLM 产出）

- **7. 专辑名称**：默认与歌曲同名；特殊情况可设计独立专辑名（分卷 Vol.X 模式见 SOP）。
- **8. 专辑描述**：不介绍歌曲、不写创作说明；优先散文诗 / 现代诗 / 短句文学表达；**20~80 字**。
- **9. 封面设计**：提供原图 / 人物设定图 / 风格参考图 → 产出封面创意方案 + MJ Prompt + 排版方案（含标题位置、字体风格、作者署名）。

---

## 六、发布后动作

- **10. 评论区引导**：产出一条置顶评论，收集真实反馈。
  - 示例：「如果这首歌让你想起某个人、某个傍晚、某间教室，欢迎把故事留在评论区。」
- **11. MV 筹备**：从听众评论中提炼真实故事——收集高频场景 / 高频情绪 / 高频人物关系，用于后续 MV 剧本开发。

## 七、发布后复盘

记录：播放量 / 收藏量 / 评论量 / 分享量。

重点关注：哪些歌词被反复提及、哪些情绪最容易引发共鸣、哪些故事最值得被拍成 MV。

---

## 给 LLM 的输入（每次发布直接提供）

下次发布新歌时，直接提供以下 5 项，LLM 按本 Brief 自动补全：

1. 歌曲名称
2. 歌词
3. 音频链接（可选）
4. 封面素材（可选）
5. 希望的参考风格

自动补全：标签 / 主创说 / 站内笔记 / 专辑描述 / 封面方案 / MV 方向 / 发布后互动策略。
